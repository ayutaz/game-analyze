## なぜ EXP 主分類なのか

プレイヤーの主動機は「自分で空間を作る／作るために掘る」という創造体験そのものである。コンセプト文（「ブロックを採掘して何でも作るサンドボックスの代名詞」）と target（全年齢、創造性を発揮したい層、教育用途）が示すように、報酬や物語ではなく **行為の連鎖そのものが目的化** している点が EXP 分類の根拠となる。

social_axis は「協力」。Realms / Java マルチサーバ / 統合版フレンドワールドを通じて、他者と同じ空間を共有しながら役割分担して建築・採掘・冒険する遊びが中核を占める。一方でソロでも完結し、シード値ベースの単独サバイバルも成立する。secondary は空配列で、副次動機を意図的に薄めた **行為一元の設計** になっている。報酬要素（実績、エンチャント、エンドラ討伐）は存在するが、それらは creative mode で全部スキップしても遊びが壊れない、つまり主動機ではないと運営自身が宣言している構造である。

## 体験の核 / 設計の発明

| 設計要素 | 仕様 | プレイヤーへの効果 |
|---|---|---|
| ボクセル地形 | 1m³ ブロック単位で破壊・設置 | 全空間が編集対象になり「作る／壊す」が地続き |
| 無限プロシージャル世界 | シードから無限生成 | コンテンツ枯渇の概念がなくなる |
| クラフトレシピ | 2D グリッド配置で道具を合成 | 探索 → 採掘 → 強化のループが自走 |
| Redstone | ブロック単位の回路素子 | ゲーム内に計算機・自動化機構を作れる |
| MOD / データパック | Java 版で外部拡張可 | プレイヤーがコンテンツ供給側に回る |

前世代の sandbox（SimCity, RollerCoaster Tycoon）が「決められた箱の中の最適化」だったのに対し、Minecraft は **地形そのものをプレイヤーが書き換えられる**点が断絶。協力軸では「分業効率」が直接ワールドの成長速度に効くため、サーバ運営・建築コミュニティ・配信者集団といった社会的ユニットが自然発生した。

## バズ要因 / 人気の根拠

- 売上は world 3億+ コピーで、単体ゲームソフトとして史上最高記録（テトリス系を除く）
- 教育版 Minecraft Education が各国の学校カリキュラムに採用され、ゲームが学習ツールへ越境
- YouTube・ニコニコでの実況・建築動画文化が拡大の主動力。ヒカキンら大手配信者の継続コンテンツ化
- MOD 文化（Forge, Fabric, Bedrock Add-on）が二次創作の経済圏を作り、本体寿命を15年規模に延ばした
- 2025 年公開の実写映画版が国内外で話題化し、IP としての横展開が継続

## 企画への示唆

- **行為を一次目的化すると寿命が伸びる**：報酬を主軸にせず「作る・掘る」自体を快感にすると、運営側コンテンツ供給に依存しないリテンションが生まれる
- **編集権限をプレイヤーに渡すと、コンテンツ供給が外部化する**：MOD / サーバ / 建築コミュニティが本体の代わりに新規体験を生産する
- **協力軸を「分業効率が直接見える」設計にすると、社会的ユニットが自然形成される**：建築班・採掘班・PvP班 が役割分化する
- **教育・公共領域に滲み出る間口を作ると、購買層の年齢上限が消える**：保護者・教師経由の購入導線が立つ
- **グラフィック解像度を意図的に低くすると、プレイヤーの想像力で補完される**：低スペック機種カバーと寿命延長を同時に達成

## 比較で見るべきタイトル

- Roblox（同じ「プレイヤーが供給側に回る」設計、ただし UGC プラットフォーム寄り）
- Terraria（2D サンドボックス、探索 → 採掘 → 強化のループは共通）
- どうぶつの森 シリーズ（非同期協力サンドボックス、創造より生活演出寄り）
- Garry's Mod / Dreams Universe（編集ツール一体型の前例・後続）
- Valheim（協力サバイバル建築、Minecraft 以降世代の精緻化）
