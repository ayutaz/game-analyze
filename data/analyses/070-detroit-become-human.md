# デトロイト ビカム ヒューマン (Detroit: Become Human)

## なぜ NAR 主分類なのか

プレイヤーの主動機は「アンドロイドの自由意志を巡る分岐ドラマを読む」こと。操作は QTE と探索が中心で、戦闘や育成のシステム的深度はほとんどない。コンセプトに掲げられた「分岐するシナリオ」が体験のほぼ全体を占めており、`concept`（自由意志を問うインタラクティブ・ドラマ）と `target`（20-40 代、映画的物語を体験したい層、哲学・倫理を求める層）の整合からも、本作は「遊ぶ」より「読む／演じる」に重心がある。

`social_axis` が空欄なのは、シングルプレイのインタラクティブ・ムービー（`genre`）で他者軸が成立しないため。`secondary` も空欄で、副次的な動機（収集・対戦・育成）がほぼ存在しない構造のシンプルさが、NAR への純度を高めている。

## 物語の駆動構造

3 主人公（コナー / カーラ / マーカス）の章を交互に進行させ、各章末に **フローチャート画面で「自分が辿らなかった分岐」を可視化** する仕掛けが、再プレイと SNS 共有を同時に駆動する。

| 構造要素 | 役割 | 効果 |
|---|---|---|
| 3 視点並行 | 警察捜査 / 家族逃亡 / 革命指導 の異質な物語を交差させる | テーマ（自由意志）を多角的に検証 |
| キャラクター死亡分岐 | 主人公すら章中で恒久退場する | 1 周の重み・選択の責任感を担保 |
| フローチャート可視化 | 未踏ルートをノード単位で表示 | 「埋めたい」欲求が周回・実況視聴を生む |
| QTE と探索の薄さ | 操作難度を意図的に下げる | 物語消費に集中させ、非ゲーマー層も取り込む |

Quantic Dream が『Heavy Rain』（2010）で確立した「QTE で映画を分岐させる」骨格を、UI 設計（フローチャート）で **体系化・可視化** したのが本作の発明。プレイヤーは「物語を読む」と同時に「分岐ツリーを完成させる」というメタな動機を持つ。

## バズ要因 / 人気の根拠

- 全世界 950 万本超のセールスは、ナラティブ単体作品としては突出した規模。シリーズ続編なしでこの数字に到達している。
- フローチャート UI が **「他人の選択と比較する」配信文化** と相性が良く、YouTube / ニコニコでの実況プレイが多数派生。視聴者が「自分ならこうした」とコメントで参加できる構造。
- 「アンドロイドに人権はあるか」というテーマが公民権運動・AI 倫理・移民問題のメタファーとして読解可能で、ゲームメディア以外（社会派の文脈）にもレビューが波及。
- PS4 独占（後に PC 移植）かつ Quantic Dream のオリジナル IP として、ソニーが映画的体験を象徴する旗艦タイトルとして長期間プロモーションした。

## 企画への示唆

- **未踏ルートを可視化すると、1 周クリア後の継続動機が生まれる**：フローチャート / 達成率 UI を物語ゲームに載せると周回率と実況再生数が連動して伸びる。
- **主人公を恒久退場させると、選択の重みが体感値として戻る**：「失敗してもやり直せる」前提を壊すと、1 つ 1 つの QTE に緊張が宿る。
- **複数視点を時間軸で交差させると、テーマの抽象度を高められる**：同じ問い（自由意志）を異なる立場で 3 度反復することで、プレイヤー自身に解釈の余地を委ねる構造になる。
- **操作難度を下げると、非ゲーマー層が物語の入口に立てる**：QTE 中心設計は熟練度の壁を消し、「映画を観たい層」を取り込む受け皿になる。
- **社会的論点を物語の前提に置くと、ゲーム外メディアでの言及が増える**：AI 倫理・公民権など現実の議論にひも付くテーマは、ゲーム文脈外のレビュー流入を呼び込む。

## 比較で見るべきタイトル

- **Heavy Rain (2010, Quantic Dream)** — 同スタジオの前世代。QTE 分岐の原型だがフローチャート可視化はなく、本作で UI 化された。
- **十三機兵防衛圏 (2019)** — 13 人視点・時系列交差の NAR 設計。視点切替の頻度設計を比較する素材として有用。
- **ライフ イズ ストレンジ (2015)** — 章末で選択結果を可視化する文化を北米インディーから持ち込んだ系譜上の隣接作品。
- **428 〜封鎖された渋谷で〜 (2008)** — 国産マルチサイトクリア型 ADV。フローチャートと TIP を駆使する構造の元祖の一つ。
