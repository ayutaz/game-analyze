# ハンドレッドソウル / Hundred Soul

## なぜ REW 主分類なのか

このゲームを遊び続けるプレイヤーの主動機は「装備を掘る・Fellow を集める・育てる」である。Hound 13 が Unity 上で組んだコンソール品質のアクションは強い掴みだが、ランタイムの設計はガチャ・装備強化・Fellow 強化を縦に積んだ長期運営型モバイル ARPG であり、リテンションを支えるのはステージ周回によるドロップとガチャである。

`concept` 上も「100 体以上の装備と Fellow を切り替える」「装備掘り・ガチャ・育成ループでプレイヤーを長期につなぎ留める」と明示されており、`target` の「装備ガチャ＋やり込み志向」がそのまま主分類を裏付ける。`social_axis` は非同期で、ギルド・PvP ランキング・周回効率の競争が他者比較として効くものの、卓を囲む対戦体験ではなく「自分のビルドが他者比較でどこに立っているか」を見せる軸である。

副次的な EXP（アクション操作の手触り）は、グラフィックと打撃感で「ハイエンド ARPG として恥ずかしくない」体感を維持し、報酬ループが空虚に見えないための土台になっている。アクションが副、報酬が主、という重み付けが REW 判定の根拠である。

## 報酬ループの組成

| 層 | 中身 | リテンションへの効き方 |
|---|---|---|
| ガチャ | Fellow（仲間キャラ）／装備（コアの掘り対象） | 新規ピックアップで山場、SSR 装備の OP 差で復帰動機 |
| 周回 | ステージドロップで素材・装備の母数を増やす | 日課としての滞在時間を稼ぐ |
| 育成 | 装備強化・覚醒、Fellow レベル・スキル強化 | 引いた素材を「自分のビルド」に変換する作業層 |
| 競争 | PvP ランキング、ギルド、レイド | 育てたビルドを「他人比較」で確認する出口 |

ガチャ → 周回で素材確保 → 育成で価値変換 → ランキングで承認、という 4 段が縦に積まれ、ループのどこかで詰まっても別の層に逃げられる構造になっている。コンソール級アクションを謳う一方、操作の腕でビルド差を覆しにくい数値設計に寄っており、その分「装備をどれだけ揃え、強化を進めたか」が PvP・レイドの結果に直結する。これは韓国系ハイエンド ARPG（七つの大罪、HIT、Black Desert Mobile）と共通する設計思想である。

## バズ要因 / 人気の根拠

- 韓国先行リリースで 1.6M DL 超（Unity blog 2020）に到達し、地域限定タイトルとしては大きな初動を作った
- 2019 年のグローバル展開時に Apple／Google 両ストアの Editor's Choice を獲得し、「ハイエンドモバイル ARPG の代表格」というプラットフォーム公認の地位を得た
- Unity の HDR レンダリングと精緻なキャラクターアニメで「コンソール級グラフィック」を打ち出し、技術ショーケースとしても語られた（Unity 公式ブログが事例として取り上げ）
- 2023 年に『Hundred Soul: The Last Savior』としてリブートされ、IP 単発で終わらず継続運営の系譜に乗った

## 企画への示唆

- 「コンソール級グラフィック」を打ち出すと、モバイル ARPG のレッドオーシャンでも初動の取材露出とストア特集を取りやすい
- アクションを主訴求にしても、課金導線・リテンション設計は装備ガチャ＋育成ループに任せると運営コストが下がる
- Fellow（仲間）と装備の二軸ガチャにすると、キャラ IP と数値強化の両方をピックアップ対象にでき、新規ピックアップ周期を短く保てる
- 非同期 PvP とギルドを「育成成果の出口」として明示すると、周回・強化の作業層に意味が戻る
- リブート前提でブランドを設計すると（数年運用 → 同名タイトルの再投入）、サ終時のユーザー資産を次プロジェクトへ繋ぎ直しやすい
