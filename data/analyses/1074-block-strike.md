# Block Strike (2015 / Rexet Studio)

## なぜ EXP 主分類なのか

プレイヤーの主動機は「ロースペック端末でカジュアルに撃ち合う」ことであり、対戦の体験そのものが滞在時間を作っている。`social_axis: 対戦` のとおり Team Deathmatch を中心とする PvP がコアループで、スキン収集はあくまでセッション継続のための周辺装置だ。

`concept` が言う「ブロック調のローポリ世界で 20+ モードを遊べる軽量・カジュアル寄りのマルチプレイヤー対戦 FPS」は、Counter-Strike 系の撃ち合い体験を Minecraft 風の親しみやすい表層に載せ替えた構成で、target に明示される「Minecraft 世代のティーン〜20 代男性／ロースペック端末ユーザー」に体験の敷居を下げて届けている。secondary に置かれた REW は、武器・キャラスキンとクラン機能でセッションを跨いだリテンションを担う副軸で、主動機を置換するほどの重みは持たない。

## 体験の核 / 設計の発明

中核は「Minecraft の表層 × Counter-Strike の中身」というジャンル接合である。グラフィックの簡素化はそのまま (a) 描画負荷の低下 = 普及帯 Android での 60fps、(b) ヒットボックスの単純化 = 初心者でも当たる射撃感、(c) アセット制作コストの低下 = モード／マップを増やしやすい体質、という三重の効果を生む。

| 設計要素 | 内容 | 体験への効果 |
|---|---|---|
| 表層 | ブロック調ローポリ | ロースペック端末で動作、Minecraft 世代に親和 |
| 射撃系 | CS 系の TDM／爆破系 | 既知の文法で学習コストゼロ |
| モード幅 | 20+（TDM、ゾンビサバイバル他） | 飽きの遅延、別動機ユーザーの吸収 |
| マップ幅 | 50+ | セッション毎の新鮮さ |
| 収集軸 | 武器・キャラスキン | 対戦の外側にリテンションを敷く |
| コミュニティ | クラン機能 | 個人プレイを社会的滞在に変換 |

前世代の Mini Militia や Pixel Gun 3D が築いた「モバイル簡略 FPS」の系譜上で、Block Strike はモード数とマップ数の物量で差を作り、`social_axis: 対戦` を 20+ モードに分解することで対戦疲労を逃がしている。

## バズ要因 / 人気の根拠

- 世界累計 **50M+ DL** という規模は、Rexet Studio という小規模インディースタジオの作品としては突出している
- iOS / Android に加え PC / Mac / Linux までクロス展開し、入口を広く取った（モバイル単独運営ではない）
- 「Minecraft 風 × FPS」というキーワードはストア検索・YouTube 検索の両方で強く、Minecraft 系コンテンツに飢えた層のオーバーフロー需要を吸収した
- 20+ モード／50+ マップというボリュームが、YouTube／TikTok のショート動画で「こんなモードもある」式の発見コンテンツを継続的に供給した

## 企画への示唆

- 既存ジャンルを **見た目だけ別ジャンルの表層に載せ替える** と、ジャンル未経験層を取り込みつつ既プレイ層には「軽量版」として刺さる
- ローポリ化は単なる省力ではなく、**描画負荷・ヒット判定・制作コスト** の 3 つを同時に解く設計レバーになる
- モバイル PvP の飽き対策には、難易度ではなく **モード数の物量** で対戦疲労を逃がすのが効く
- 主動機（対戦）に対し収集軸を **副軸として薄く敷く** と、ガチャ依存に陥らずリテンションだけ取り出せる
- 小規模インディーでも **クロスプラットフォーム配信** で母集団を 5,000 万 DL 級に押し上げ得る、という規模感の証明になっている

## 比較で見るべきタイトル

- Pixel Gun 3D（同じ「ブロック調 × モバイル FPS」前世代）
- Standoff 2（モバイル CS クローンの直接競合）
- Critical Ops（モバイル CS 系のもう一つの系譜）
- Counter-Strike 2（参照元としての PC 競技 FPS）
- Minecraft（表層意匠の源流、ターゲット層の重なり）
