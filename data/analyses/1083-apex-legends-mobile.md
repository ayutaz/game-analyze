# Apex Legends Mobile — 詳細分析

## なぜ EXP 主分類なのか

プレイヤーの主動機は「3 人スクワッドでレジェンドのスキルを噛み合わせて撃ち勝つ」ことであり、動詞は **競う／戦う** に集約される。報酬（バトルパス・スキン・キャラクター解放）は副次的な滞在動機に過ぎず、毎セッションの満足は **対戦の勝敗とアビリティ連携の手触り** から生まれる。

`social_axis` は当然「対戦」。本家 Apex Legends がスクワッド単位の即時 PvP を中核に置くタイトルである以上、そのスマホ移植も同じ軸を継承する。`secondary: REW` は、F2P モバイルとして避けて通れない **バトルパス / キャラクター解放 / スキンガチャ** という収益ループを指しており、ここが弱いと運営が成立しないが、プレイヤーが「Apex を遊んでいる」と感じる中心ではない。`concept` が明示する「タッチ操作対応」「短時間マッチ」も、体験の輸送（PC の手触りをモバイルに移す）を最優先したことの傍証である。

## 体験の核 / 設計の発明

中核メカニクスは **「レジェンド固有スキル × 3 人スクワッド × 短時間 BR マッチ」** という Apex の三層構造を、タッチ UI と回線で成立する範囲に圧縮した点にある。前世代（PUBG Mobile / CoD Mobile）との差分を要素ごとに分解すると次の通り。

| 設計要素 | PUBG Mobile / CoD Mobile | Apex Legends Mobile |
|---|---|---|
| 個性 | 武器・装備中心、キャラは皮 | レジェンド固有アビリティ＋アルティメット |
| 編成 | スクワッド4 or デュオ | 3 人固定スクワッド（本家準拠） |
| 復活 | ノックダウン→蘇生 | バナー回収＋リスポーンビーコン |
| マッチ長 | 25〜30 分 | 短縮版マップで 15〜20 分 |
| 移植戦略 | 別タイトルとして再設計 | 本家の核を残しつつ操作だけ再構築 |

「対戦」軸への効果は、**スキル発動を含む差し合いがモバイル FPS に持ち込まれた** こと。エイム力一辺倒だった既存モバイル BR に「タイミング系アビリティの読み合い」を持ち込み、初月の話題量と Apple/Google の Game of the Year 受賞につながった。

## バズ要因 / 人気の根拠

- `popularity` の通り 2022/5 世界配信、初月 **21.8M DL** を記録（mobilegamer.biz）
- 累計 **37M+ DL / 約 $40M 収益** に到達するも、収益カーブが急減し 2023/5 にサービス終了（Sensor Tower / Famitsu）
- Apple App Store / Google Play の **2022 Game of the Year** を同時受賞、品質評価は高かった
- 本家 Apex のキャラクター・武器・マップを忠実移植したことで、PC/家庭用プレイヤーが「もう一画面」として触る導線が機能した

## 企画への示唆

- **本家 IP の核（ここでは「レジェンドのスキル連携」）を削らずに操作系だけ作り直すと**、初動の信頼が稼げる（既存ファンが翻訳役になる）
- **ヒーローシューターを BR に被せると**、エイム力の差が縮まり、ライト層が「アビリティで勝てた」という別軸の達成感を得る
- **モバイル F2P で本家と同じスキン経済を再現すると**、ARPU は本家プレイヤーが既に PC 側で消費している分、頭打ちになりやすい（収益急減の主因仮説）
- **Apple/Google の GotY を取っても**、3〜6 ヶ月後のリテンションが想定を下回ればパブリッシャー判断でサ終に至る（品質と継続収益は別物）
- **Tencent 子会社の Lightspeed & Quantum × EA という二社体制は**、開発速度と本家整合性のトレードオフを生む。運営判断の分岐点でどちらの収益基準を採るかが寿命を決める

## 比較で見るべきタイトル

- PUBG Mobile（同 BR・対戦、Tencent 系の長期運営成功例）
- Call of Duty: Mobile（同 FPS・対戦、IP 移植 + 別ゲーム化戦略）
- Fortnite Mobile（BR・対戦、App Store 排除後も自社配信で延命）
- Apex Legends（本家、PC/家庭用 EXP・対戦）
- Battlefield Mobile（同じく EA × 外部スタジオ、リリース前にキャンセル）
