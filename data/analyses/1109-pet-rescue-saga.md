# ペットレスキュー サーガ (Pet Rescue Saga)

## なぜ REW 主分類なのか

プレイヤーの主動機は「ステージを攻略してマップを埋め、星と進行度を集める」こと。パズル自体を解く手応えよりも、**サーガ式マップを前に進めること自体が報酬**になる設計で、King の Candy Crush 系と同じ構造を共有する。

concept にある「手数制限 × ステージクリア型 × ブースター課金 × フレンド競争」は、典型的なリテンション設計だ。一手ごとに「クリア＝次のステージ解放」「失敗＝ハート消費」という二項のフィードバックを返し、進行マップというメタ報酬で長期束縛する。target が 20–50 代女性・非ゲーマー層であることも、操作の熟達ではなく「進む」ことを報酬として供給するゲーム性と整合する。

secondary に EXP が入るのは、1 ステージ単位ではブロック消しの認知的快感（マッチ系特有の連鎖手触り）が確かに残るためで、純粋な収集ゲーではない。ただし主たる継続駆動力は「マップを埋める」報酬ループ側にある。

## 報酬ループの組成

| 層 | 提供物 | リテンションへの効き方 |
|---|---|---|
| 即時報酬 | ブロック連鎖、ペット救出演出、星 1〜3 評価 | 1 ステージ 1〜3 分のドーパミン |
| 短期目標 | 次のステージ解放、エピソード踏破 | 「あと 1 ステージだけ」の連打 |
| 長期目標 | サーガマップの踏破、新エピソード追加 | 数年単位の更新サイクル |
| 経済層 | ハート（ライフ）回復待ち、ブースター、追加手数 | フレンドからのハート寄付 + ストア課金 |
| 社会層 | Facebook 連携でのフレンド進行可視化、ランキング | 「友達より先に進みたい」競争動機 |

中核の発明は、(a) ハート式ライフによる**プレイ機会の希少化**、(b) サーガマップによる**進行の可視化**、(c) フレンドの位置がマップ上に表示される**横並び競争**、の三点セット。Candy Crush で完成した型を、可愛いペット救出という別 IP に被せて再利用した。難易度カーブも前半は易しく、ある区間から急に詰まり、ブースター課金または数日待機を促す山岳構造で、F2P 経済の標準形を踏襲している。

## バズ要因 / 人気の根拠

- 累計 **2 億 DL 超**（sales_world: 210M+ DL）。Candy Crush Saga ほどではないが、King のサーガ系セカンドヒットとして長寿運営を続けている。
- 「ブロックを消す」だけでなく「**かわいいペットを救出する**」という情緒的フックを足し、純粋な数字パズルより SNS 上での共有・口コミに乗りやすい絵面を持つ。
- Facebook 連携によりフレンド進行がマップ上に並列表示され、**横並び型の競争**がカジュアル層の継続を駆動。ハートの送り合いがコミュニケーション化した。
- 2013 年リリースから 10 年以上にわたりエピソード追加が続き、iOS / Android / Browser / PC（Microsoft Store）でクロス配信、隙間時間アプリの定番枠を確保した。

## 企画への示唆

- **既存のヒット型に別 IP を被せる** と、ゼロから経済設計し直さずに別客層を取りに行ける（Candy Crush → Pet Rescue → Farm Heroes の King 量産ライン）。
- **ライフ制 + 待ち時間 + フレンド寄付** をセットで組むと、課金しないユーザーも SNS 内でコミュニケーション化し、自然な口コミの母集団になる。
- **サーガマップ可視化** は、進行度というメタ報酬を提供しつつ、フレンドの位置を並べることで競争動機まで束ねられる。テキスト UI のレベル一覧より強い。
- **救出される対象（ペット）** を演出の中心に置くと、純粋な数値パズルでは届かない非ゲーマー層・女性層に絵面で訴求できる。
- **3 分のステージ単位 × 数百〜千ステージ** の構造は、隙間時間アプリの再訪頻度を最大化する黄金比として再現性が高い。
