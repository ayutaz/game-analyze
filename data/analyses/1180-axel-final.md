# アクスル：ファイナル (Axel: Final) — 分析メモ

> **注意**: このエントリは出典が未確定の暫定掲載タイトルである。WebSearch (英語・日本語) では同名タイトルの一次情報を確認できず、Nexon 公式の公開ゲーム一覧にも該当が見つからない。以下の分析は **JSON に書かれた最小限のメタ情報 (developer=Nexon, year=2023, genre=アクション, primary=EXP, social_axis=ソロ)** から構造的に推論できる範囲に限定する。固有名詞ベースの記述は意図的に避けている。

## なぜ EXP 主分類なのか

プレイヤーの主動機は「操作してアクションを成立させる過程そのものを味わう」と仮置きできる。concept は "Nexon が手掛けたとされるアクション系タイトル" とのみ書かれており、target も "ネクソン作品のアクション系プレイヤー層" 止まりだが、genre=アクション × social_axis=ソロ という 2 つの確定情報からは「他者との関わりを介さず、入力 → 応答のフィードバックループで完結する体験」という骨格が読み取れる。

secondary が空配列であることも EXP 寄りの傍証になる。物語消費 (NAR) や収集・ガチャ (REW) を副次動機として明示していないということは、少なくともメタ情報の起票時点では "アクションの手触り" 以外の引きを主張していないということだ。ただし出典が確定していない以上、social_axis=ソロ も "情報なしのデフォルト値" の可能性が残る点には留意したい。

## 体験の核 / 設計の発明

確定情報が乏しいため、ここでは「Nexon × PC × ソロアクション 2023」という枠が **構造的にどの設計空間に位置するか** を整理する。

| 軸 | このタイトル | 同枠の典型例（Nexon 系列 / ソロアクション系） |
|---|---|---|
| プラットフォーム | PC 単独 | Mabinogi Heroes / DNF Duel 等は複数プラットフォーム展開が多い |
| social_axis | ソロ | Nexon の収益主軸は MMO・PvP の "対戦/協力" が中心で、純ソロは少数派 |
| 想定リテンション | 不明（出典なし） | F2P MMO 路線なら長期、買い切りソロアクションなら短〜中期 |
| 主要メカニクス候補 | アクション操作 | ハクスラ / 死にゲー / コンボアクション / Roguelike 等いずれの可能性も残る |

つまり Nexon 本体のポートフォリオから見ると "ソロ PC アクション" は周縁領域であり、もし実在するならば既存 IP のスピンオフ、もしくは社内スタジオによる小〜中規模実験プロジェクトのいずれかに位置する可能性が高い。

## バズ要因 / 人気の根拠

- popularity 欄に「WebSearch では同名タイトルの一次情報を確認できず」と明示されているため、**現時点で観測できているバズは存在しない**
- sales_jp = "—" / sales_world = "unknown" で、売上・DL の定量データもない
- _source_urls は Nexon コーポレート / 4Gamer publisher ページ / Wikipedia の 3 件にとどまり、いずれもタイトル固有の言及を含まない
- 同名・類似名の混同（プロジェクト名と最終的なリリース名のズレ、開発中止、別地域限定の小規模リリース等）の可能性も排除できていない

## 企画への示唆

このタイトル自体からは設計の教訓を引き出しにくいが、**カタログ運用の側から学べる失敗例** として 3 点を整理する。

- 出典 URL が "コーポレートサイトのトップ" レベルしか取れないタイトルを掲載すると、後続の分類精度が一気に落ちる → enrich 段階で **タイトル固有の一次出典 (公式 LP / プレスリリース / Steam ページ) が 1 件もない場合は採用を見送る** ルールが要る
- developer=publisher の self-published 判定だけでは "実在性" を担保できない → indie タグ付与とは別軸で **"出典 confidence" メタフィールド** を持つと、暫定枠と確定枠を分離できる
- social_axis=ソロ をデフォルト値として埋めると、後段の体験軸ファセットを汚染する → **未確定タイトルは social_axis を null 許容**にし、ファセット集計から外す運用に寄せる方が健全
- popularity 欄に「出典未確定」と書かれているレコードは UI 上で警告バッジを出す、もしくは詳細ページに「出典精査中」のステータスを表示する設計が望ましい
