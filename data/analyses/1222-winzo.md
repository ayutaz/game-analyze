# WinZO — 詳細分析

## なぜ EXP 主分類なのか

プレイヤーの主動機は **「familiar なボード／カジュアルで他人と腕を競い、少額の賞金を取りに行く」** こと。ゲーム自体を遊ぶ過程と PvP 対戦結果がそのまま価値になるため、報酬は二次的なフックに過ぎず体験型として分類した。

WinZO の concept は「インド版 Skillz」、すなわち Ludo / Carrom / Fantasy Cricket など既に身体化されたゲームを小さなエントリーフィー＋現金賞金のトーナメントに包んで提供するプラットフォームである。target が Tier 2/3 都市の Hindi/Bhojpuri/Tamil ネイティブを含む 18-35 歳である点も、新規ゲーム体験を学ぶ層ではなく「既に知っているゲームでマッチング相手を得たい」層が中核であることを示す。social_axis は **対戦**、1 対 1 / 数人卓のリアルタイム対人戦が主軸。

副次的に効くのが secondary の REW。現金賞金・リーダーボード・キャッシュバックなどの報酬要素はリテンションの土台だが、これは「ゲーム外の動機」ではなく「対戦の結果に値段が付くこと」で対戦そのものをシリアスにしている。報酬は対戦体験のエンジンであって主目的ではない。

## 体験の核 / 設計の発明

WinZO の発明は「単一ゲームを作らず、100+ のサードパーティ製スキルゲームを 1 つのウォレット／マッチング／KYC レイヤーで束ねた」点にある。プレイヤーから見れば「いつもの Ludo を 5 ルピーで対戦できる」、開発者から見れば「ユーザー獲得・決済・本人確認を肩代わりしてくれる流通網」という二面市場が成立する。

| レイヤー | 役割 | 前世代との差分 |
|---|---|---|
| ゲーム本体 | Ludo / Carrom / Fantasy Cricket など familiar タイトル | ゼロから IP を作らず既知ゲームを採用 |
| SDK / 開発者経済圏 | サードパーティが WinZO 上で配信 | 自社開発単独の MPL Pro / Dream11 と差別化 |
| マッチング & KYC | エントリーフィー徴収、年齢/地域チェック、賞金分配 | 各ゲーム個別に決済を組まずに済む |
| 言語ローカライズ | 12+ インド地方言語 UI | 英語前提の Skillz / グローバルゲームと差別化 |

social_axis の「対戦」は短時間 1 戦完結のマイクロセッション（Ludo 1 局 3-5 分）×現金リスクで濃度を上げる構造。少額でも「自分の腕で取った」という主観が生まれ、勝敗の反復が体験のコアになる。

## バズ要因 / 人気の根拠

popularity の「175M+ ユーザー、月間 50 億トランザクション、累計 1 億ドル調達」を分解すると以下が効いている。

- **言語の壁を先に潰した**: 12+ インド地方言語対応で Tier 2/3 都市の Hindi/Bhojpuri/Tamil ネイティブを早期に取り込み、英語 UI 前提の競合に対して獲得コストで優位。
- **familiar ゲームの再パッケージ**: Ludo / Carrom はインドの家庭で何十年も遊ばれているボードゲーム。学習コストゼロで「課金して対戦する」段階に直接入れる。
- **二面市場のスケール**: 100+ ゲームを束ねたカタログにより、ユーザーは 1 アプリで複数趣味をカバーし、開発者は WinZO の獲得・決済網に乗れる。
- **VC マネー**: Griffin Gaming Partners / Makers Fund などから累計 1 億ドル規模を調達し、マーケティング・KYC コンプライアンスへ厚く投資できた。

## 企画への示唆

- **既知のオフラインゲームをオンライン対戦化すると、学習コストを払わずに「現金が動く対戦」段階に入れる**（Ludo / Carrom の事例）。
- **言語ローカライズを「翻訳」ではなく「市場参入の前提条件」として扱うと、英語競合に対して獲得コストで持続的に有利になる**（12+ インド地方言語）。
- **自社開発ではなく SDK 開放 + 流通網提供にピボットすると、コンテンツ拡張のコストを開発者側に押し付けつつ、自社はマッチング・決済・KYC のインフラ価値で利益化できる**（Skillz 型二面市場）。
- **少額エントリーフィー（数ルピー〜）にすると、課金抵抗を「ゲーム購入」ではなく「対戦参加費」に置き換えられ、心理的ハードルが下がる**。
- **リアルマネーを扱うなら、KYC・年齢確認・地域別合法性のレイヤーを最初から共通インフラ化しないと、ゲーム追加のたびにコンプライアンス工数が爆発する**。

## 比較で見るべきタイトル

- **Skillz**（米国のスキルゲーム賞金プラットフォーム、WinZO の参照モデル）
- **MPL (Mobile Premier League)**（インド競合、自社開発比率が高い）
- **Dream11**（インドのファンタジースポーツ最大手、現金賞金 PvP の先行成功例）
- **Ludo King**（Gametion 製、インドの Ludo モバイル定番、対戦は無料・WinZO とは収益化モデルが対照的）
