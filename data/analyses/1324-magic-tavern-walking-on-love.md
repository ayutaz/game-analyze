# MR. LOVE: DREAM DATE / 恋与制作人（Magic Tavern, Walking on Love） 分析

## なぜ REW 主分類なのか

プレイヤーの主動機は「推しキャラのカード／ボイス／メモリーを引き当てて手元に揃えること」である。物語は確かに本作の魅力の中核だが、その物語シーンの大半は **カードガチャで解放される演出ユニット** として配信されており、進行のためには天井・ピックアップ・覚醒素材といった報酬ループを回し続ける必要がある。concept にある「カードガチャでイベントを解放し物語を進める、ライブサービス型乙女ゲーム」という設計が、本作を NAR ではなく REW として読むべき根拠そのものになっている。

target が「スマホで完結する手軽な恋愛シミュレーション・ボイスドラマを好む層」「アニメや CV 人気声優目当ての二次元コンテンツファン」と定義されている点も重要で、消費対象は「物語の続き」というより「特定キャラの新規ボイス・新規イラスト・新規衣装」の獲得である。secondary に NAR が入っているのは、メインストーリーと各キャラ個別ルートが課金導線の **動機形成装置** として強く機能しているためで、ガチャ単体では成立しない構造になっている。social_axis が null なのは、本質的にソロプレイの恋愛体験でありつつ、SNS でのコレクション自慢・実況・公式生配信視聴が補助的に回るため単一軸に置きづらいことを反映している。

## 報酬ループの組成

乙女向けライブサービスとして、本作は「物語消費」「キャラ所有」「ソーシャル投影」の 3 層を別々のループとして組み上げている。

| 層 | プレイヤー動機 | 主要メカニクス | 課金導線 |
|---|---|---|---|
| 物語消費 | 推しの新規シーンを読みたい | メインストーリー＋キャラ別ルート、期間限定イベントストーリー | スタミナ回復 / イベント加速アイテム |
| キャラ所有 | カード・衣装・ボイスを揃えたい | カードガチャ（SSR ピックアップ）、覚醒・限界突破 | 有償石・月パス・覚醒素材パック |
| ソーシャル投影 | 推しと「繋がっている」演出を維持したい | 電話・メッセージ・ボイスメッセージ UI、誕生日演出、写真フレーム | 衣装課金・誕生日限定パッケージ |

特徴的なのは三層目で、ホーム画面でキャラから電話がかかってきたり、リアルタイムに近いメッセージ通知が届いたりするスマホネイティブな UI を採用し、所有したキャラが「日常に存在する」感覚を演出している。これにより、ガチャで引いたカードが単なるステータス上昇ではなく、 **日常通知の解像度** に変換される。カードバトル自体は属性相性のシンプルな自動戦闘で、戦闘の戦略性よりも「強い推しを持っているか」という所有確認の儀式として機能している。

## バズ要因 / 人気の根拠

- 中国本土版『恋与制作人』が 2018 年 1 月に約 32M USD を 1 か月で売り上げ、女性向けスマホ恋愛ゲームの収益スケールを一気に塗り替えた。
- ガチャ＋ボイス＋疑似電話・メッセージ UI を組み合わせた「ライブサービス型乙女」のテンプレートを確立し、後続の中華乙女・グローバル乙女タイトルの参照点となった。
- 2020 年に MAPPA 制作のアニメ版が放映され、二次元コンテンツ層・声優ファン層に向けた認知導線が形成された。
- 海外版 Mr Love: Dream Date として英語圏・東南アジアで継続運営されており、Sensor Tower 上でも US 市場で長期に渡り乙女カテゴリの上位常連として可視化されている。

## 企画への示唆

- ガチャの報酬を「ステータス値」ではなく「日常通知の解像度」に変換すると、SSR を引く動機が戦闘有利ではなくキャラとの関係維持に置き換わる。
- 電話・メッセージ・ボイスメモといった OS ネイティブ UI を模倣すると、ゲームを起動していない時間にも所有感を持続させられる。
- 物語をメイン一本ではなくキャラ別ルートに分割しイベント毎に小出しにすると、推し単位での課金動機がイベントごとに再点火する。
- カードバトルを自動・短時間化すると、戦闘がスキップ可能な「所有確認の儀式」に縮約され、テキスト・ボイス消費に時間予算を再配分できる。
- 公式アニメ化など他メディア展開を前提に IP を設計しておくと、サービス中盤以降の新規ユーザー獲得チャネルを多重化できる。
