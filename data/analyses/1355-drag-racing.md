# Drag Racing (Classic) — 詳細分析

## なぜ EXP 主分類なのか

プレイヤーの主動機は「指のタイミングだけで他人より速くゴールする」こと、つまり**競う**にある。1 レース 10〜20 秒の極短セッションで、スタートの RPM 合わせ・シフトアップの瞬間・ニトロ投入の 3 操作だけで勝敗が確定する設計は、報酬を取りに行く動機よりも「次は 0.05 秒削れる」という身体反射の改善体験が中核に据えられている。

concept にある通り対象操作は 3 つに圧縮されており、target は「隙間時間で達成感」を求める層。社会軸は **対戦**で、非同期ゴーストと、後に追加されたリアルタイム 1v1（Pro League）の両方が、世界中の同チューニング車との比較を成立させる。チューニング・車両収集は明確に存在するが、それは「相手と互角に競うための装備調整」であって、収集自体を目的化しない。したがって主分類は EXP、secondary に REW（車両アンロックと部品強化のループ）が乗る構造として読める。

## 体験の核 / 設計の発明

3 操作のみという入力削減と、ライセンス実車のチューニング深度を両立させた点が本作の中核発明。前世代のレースゲームが「コーナリング技術＋周回」で差別化していた中、Drag Racing は直線一本に限定して**反応速度ゲームに変換**し、スマホのタッチ UI に最適化した。

| 設計要素 | 旧世代レース（コンソール） | Drag Racing |
|---|---|---|
| セッション長 | 3〜10 分／周回 | 10〜20 秒／1 レース |
| 入力数 | ステア＋アクセル＋ブレーキ＋ギア＋ドリフト | RPM 待機＋シフト＋ニトロ |
| 勝敗要因 | ライン取り＋車種＋技量 | 反応 ms＋セッティング数値 |
| 対戦軸 | 同期マルチ／タイム比較 | 非同期ゴースト＋RT 1v1 |
| 収益化 | 買い切り＋DLC | 部品アンロック／車両ガチャ的解禁 |

social_axis の「対戦」は、純粋な PvP ではなく**チューニング数値の比較戦**として機能している。同クラス（A〜S クラス）にマッチングを限定することで、調整努力がそのまま勝率に反映される設計を担保した。

## バズ要因 / 人気の根拠

- 全世界 150M+ DL（公式公称、シリーズ累計）。スマホ黎明期 2010 年リリースで初代スマホレース定番枠を取り切った
- NY Times の 2011 年ベスト 10 ゲーム選出、Webby Awards モバイル部門入賞といった**西側メディア承認**が初期ユーザー獲得を加速
- 50 種以上の**ライセンス実車**（Nissan / BMW / Mercedes 等）を低スペック端末で動かせた技術選択。2D 横スクロールに割り切ったことで、当時の Android 低価格機にもインストールベースを広げた
- 「セッション 20 秒」が通勤・トイレ・授業の合間にハマる**マイクロセッション需要**を先取りし、同時期の Temple Run / Angry Birds と並ぶ「隙間時間ゲーム」カテゴリを形成

## 企画への示唆

- 操作を 3 つに圧縮すると、コンソール経験者と新規スマホユーザーの**スキル格差が縮まり**、対戦の母集団が太る
- セッション長を 10〜20 秒に固定すると、広告挿入位置とリトライ動機が同時に設計できる（負け→即リトライの摩擦ゼロ化）
- 「収集」を主目的にせず「対戦の装備調整」と位置付けると、課金プレッシャーが下がりつつチューニング深度を維持できる
- 同クラスマッチングを採用すると、課金者と無課金者の**勝率差を構造的に抑制**でき、長期リテンションが伸びる
- ライセンス実車 × 2D 表現の組み合わせは、IP 訴求と動作要件削減を両立する**低スペック端末攻略の定石**になる

## 比較で見るべきタイトル

- CSR Racing / CSR2（NaturalMotion）— ドラッグレース・3 操作・実車ライセンスのフォロワー、3D 高品質化路線
- Real Racing 3（EA）— 同時期スマホレース、フル 3D ＋周回サーキット路線でリッチ化に振った対極
- Asphalt 8/9（Gameloft）— アーケード派手系、ニトロ＋ジャンプで対戦体験を増幅した方向
- Temple Run / Subway Surfers — 1 セッション短尺＋反射操作という同じ「隙間時間」設計原理
