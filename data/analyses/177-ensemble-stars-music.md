# あんさんぶるスターズ!! Music

## なぜ REW 主分類なのか

プレイヤーの主動機は「推しの男性アイドル（カード／3D MV）を集めて育てる」ことである。リズムゲーム部分は楽曲消費とスコアタの導線だが、ガチャ・育成・MV 鑑賞という報酬群がプレイ時間の支配項であり、原作あんスタの物語消費（NAR）は secondary に退いている。

concept にあるとおり本作は 2020 年に既存『あんさんぶるスターズ!! Basic』と並走する形でリリースされた「リッチ版」で、Basic 側がストーリー・カード読み物中心であるのに対し、Music は 3D MV と楽曲ガチャを主軸に据えている。つまり同一 IP 内で「物語消費」と「楽曲＋カード収集」を意図的に分業させた構造で、Music を遊ぶ動機は明確に後者になる。target が「10〜30 代女性、3D MV・楽曲目当ての新規層」と記載されているのも、収集と鑑賞を主動機として設計したことの裏返しである。

secondary に NAR が立つのは、楽曲やイベントカードに紐づくユニットストーリー・読み物が常時供給され、ガチャ動機を物語が下支えしているため。社会軸は null だが、実態としては「フレンドのスコア比較」「ランキング上位入賞による限定報酬」という非同期ソーシャルが弱く存在する。

## 報酬ループの組成

| 層 | 中身 | 設計効果 |
|---|---|---|
| 楽曲レイヤー | 数百曲規模の楽曲・ユニット別 MV | 推しユニット単位の継続動機、新曲ガチャの口実 |
| カードガチャ | アイドル 40+ 名、衣装違いカード | 「推しの新衣装」が出るたびに集金イベントが立つ |
| 育成 | カード覚醒・特訓・アイドルロード | 引いた後の燃焼工程、無課金層の時間消費 |
| イベント | ランキング・採点・収集型を回転 | 月次の集中課金窓を作る |
| MV 鑑賞 | 高品質 3D MV、衣装変更可 | カード取得後の「使う」場面を提供しリテンション |

ループの妙は、ガチャ→育成→MV で衣装を着せて鑑賞→新曲告知→再ガチャ、と「収集物の使い道」が常に MV と楽曲側に回収される点にある。Basic 側が読み物で同じカードを使う構造になっているため、Music で引いたカードは双方の経済で価値を持つクロスアプリ設計でもある。

## バズ要因 / 人気の根拠

- 国内 DL 1000 万本超（IP 全体／Music 単体で公称、シリーズ累計）。男性声優アイドル IP として広く認知された規模。
- 男性アイドル系モバイル IP で「3D MV × 楽曲量」を本格投入した代表格の一つで、女性層の新規流入の起点になった。
- 原作『あんさんぶるスターズ!』（2015〜）からの継続層を持ち込めた点。新規 IP ではなく既存 IP の派生としての立ち上げ。
- Basic と Music の二アプリ並走運営により、物語勢と楽曲勢の双方を逃さず保持する構造。

## 企画への示唆

- 既存 IP の派生アプリを「物語版」「楽曲版」に分業すると、同じカード資産を両アプリの経済で二重利用でき、ユーザーあたり LTV を引き上げられる。
- ガチャの主目的を「カード」ではなく「衣装＋ MV」に転位させると、引いたあとの使い道（鑑賞）が報酬体験として独立し、消化不良が減る。
- アイドル数を 40+ に拡張すると、ガチャの当たり外れを「推し基準」で再定義できるため、SSR 偏重の不満を吸収できる。
- 楽曲を継続供給できる体制（ユニット別アルバム展開）があれば、ガチャの口実が自動的に毎月立ち、運営側の企画コストが下がる。
- Basic 側に物語、Music 側に楽曲を寄せる二段構造は、片方の更新が滞っても IP 全体の継続を担保するリスク分散にもなる。
