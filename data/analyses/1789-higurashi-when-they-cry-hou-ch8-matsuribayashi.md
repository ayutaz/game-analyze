# ひぐらしのなく頃に解 祭囃し編 (2006)

## なぜ NAR 主分類なのか

プレイヤーの主動機は明確に「読む / 真相を見届ける」である。出題編 4 章 + 解答編 3 章を経た上で本作（第 8 章）に到達するプレイヤーは、雛見沢症候群と連続怪死事件をめぐる長尺ミステリーの「最終解答」を回収するために起動する。選択肢のないサウンドノベルというフォーマット、`social_axis: null`、`secondary: []` というデータの寡黙さは、本作が他者性・収集・反復で動くゲームではなく、純粋に「物語消費を駆動する単線テキスト」であることを反映している。

`concept` にある通り、本章は「最後のループ」を描き、ジュブナイル＋ホラー＋ミステリーの三層構造を読者側に解放する役割を持つ。プレイヤーの能動性はほぼゼロで、提示される情報・伏線回収・キャラクター動機の整合をひたすら追跡することに尽きる。これは典型的な NAR（物語型）の動作モデルである。

## 物語の駆動構造

本作の駆動エンジンは「章単位でリリースされる出題編 → 解答編 → 完結章」という連載型構造そのものに埋め込まれている。プレイヤーは 1 章を読む → 考察 → 次章を待つ → 答え合わせ、というサイクルを 3 年間繰り返し、最終章で全変数が回収される。

| 構造要素 | 機能 | 祭囃し編での役割 |
|---|---|---|
| 章立て連載（同人頒布） | プレイヤーを継続的に呼び戻す | 全 8 章の終端、考察コミュニティの解散点 |
| ループ世界観 | 同一登場人物・同一日付の反復で謎の輪郭を彫る | 「最後のループ」で結末を確定 |
| 雛見沢症候群という SF 装置 | 怪奇・ホラー的事象に理屈を与える | 全章で散らした症状を一本の医学設定で束ねる |
| キャラクター関係性 | 仲間・部活メンバーが解決の駆動力になる | 「仲間と運命を変える」最終共闘の構図 |
| 章間考察フェーズ | プレイ外の読者活動を物語の一部にする | 解の前章までで張った伏線の答え合わせを供給 |

完結章という性質上、本作単体ではなく「シリーズ累積体験の最終 7〜10 時間」として設計されている点が特徴的だ。

## バズ要因 / 人気の根拠

- コミケ 70（2006 年夏）で頒布された同人ノベル本編の完結章であり、当時の同人ノベルゲーム市場の最大級ムーブメントを締めくくる「イベント」として機能した。
- 出題編から解答編までの 3 年間で蓄積された考察コミュニティが、最終章リリースに合わせて答え合わせを集中投下し、二次的な口コミ波が発生した。
- 後年、コンシューマ移植、漫画・アニメ・実写化、Steam 配信（MangaGamer 版、英語ローカライズ）へと展開し、同人発 IP がメディアミックスへ昇る代表例として参照され続けている。
- 開発元 07th Expansion が個人サークル規模で連載完結まで走り切ったこと自体が、同人ゲーム制作者の参照点となっている。

## 企画への示唆

- 完結章を「ゲーム単体」ではなく「数年に渡る連載体験の終端」として設計すると、プレイヤーは購入直後ではなく数年間の累積で物語に投資し続ける。
- 出題編 / 解答編 / 完結章という章ロールを明示的に分けて配信すると、プレイヤー間で「自分はどこまで読んだか」が共通言語化し、考察コミュニティが自然発生する。
- ループ構造で同一登場人物を反復させると、章ごとに新キャラを増やす必要がなくなり、少人数開発でも 7+ 章の連載が現実的になる。
- ホラー / 怪奇現象に SF・医学的な「合理的説明装置」を後から差し込むと、ジャンルを跨いだ二度読みが発生する（恐怖→納得→再読）。
- 同人イベント発の頒布タイミングを章リリースに紐付けると、制作スケジュール自体が販促イベント化する。
