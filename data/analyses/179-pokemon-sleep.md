# Pokemon Sleep

## なぜ REW 主分類なのか

プレイヤーの主動機は「寝ることでポケモンを集める」——つまり睡眠という生活行動を投入物にしたコレクション獲得である。動詞は「集める／育てる」であって「遊ぶ／戦う／読む」ではない。

concept にある通り、本作は睡眠計測アプリと収集ゲームを接合した構造で、ユーザーは毎晩スマホを枕元に置き、翌朝起きた時点で「寝顔ポケモン」が一定確率で出現する。target に挙がる「健康志向の 20-40 代／毎日触れる習慣型ゲーマー」が示すように、コア体験は「眠る → 朝起きてガチャ的に成果を確認する」という非同期な報酬開封である。social_axis が null なのは、対人 PvP も協力ボスもなく、報酬獲得は基本的にソロ完結するため。それでも EXP に倒せないのは、プレイヤーが操作するゲーム本体の所要時間が極小で、毎日の動機が「寝顔図鑑を埋める／カビゴンを育てる」という収集と育成にあるからだ。secondary は空欄だが、実質的には健康トラッキングという生活インフラ機能が副次効用として動いている。

## 報酬ループの組成

| レイヤー | 中身 | リテンションへの効果 |
|---|---|---|
| デイリー | 睡眠計測 → 寝顔ガチャ → 図鑑登録 | 毎晩アプリを立ち上げる物理的な動機 |
| ウィークリー | カビゴンの「ねむけパワー」育成と週末リセット | 1 週間単位のリズムを生活に固定 |
| 累積 | 寝顔図鑑（種別 × 寝顔パターン）の埋め | 数百セルのマトリクスがエンドコンテンツ |
| 課金 | プレミアムパス（プラスポイント） | 確率補正・育成効率を時短する間接課金 |

報酬ループの肝は「睡眠時間そのものを資源化」した点にある。プレイヤーが投入するのは時間とお金ではなく就寝行動で、ガチャ回数は 1 日 1 回に強制ロックされる。これがソーシャルゲーム特有の「無限ガチャ疲労」を構造的に回避し、健康改善というオフゲーム上の便益を獲得感に転化している。

## バズ要因 / 人気の根拠

- 国内 DL 1000 万本超、世界 DL 2500 万本超に到達し、健康／睡眠アプリ × IP 収集の組み合わせとして類例のない規模に至った。
- ポケモン IP 本流のバトル要素を意図的に外し、睡眠計測という「ゲーム外行動」に動機を移植した点が SNS で話題化。寝顔スクリーンショットの共有が自走拡散の核になった。
- Pokemon GO 以降の「日常生活に IP を貼る」流れの延長線上にあり、位置情報の代わりに睡眠データを使うという対称構造として理解された。
- リリース直後の話題性に加え、図鑑コンプ志向のユーザーが長期的に張り付くため、ピーク後も国内ストアランキングで安定した存在感を維持している。

## 企画への示唆

- 既存のヘビーな IP に「IP の中心メカニクスを引き算した派生作」をぶつけると、バトル疲れした既存ファンとカジュアル層を同時に拾える。
- ガチャ回数を生活行動（睡眠、歩数、起床時間）に紐づけると、課金疲労ではなく生活改善という別軸の正当化が成立する。
- 報酬解放を 1 日 1 回に強制ロックすると、可処分時間ではなく可処分日数で母数が広がり、健康志向層という「アプリゲーマーでない人口」にリーチできる。
- 寝顔のようにユーザー側がスクリーンショットで共有したくなる「収集物の見た目バリエーション」を多軸化すると、UGC で広告費を節約できる。
- IP 連結作で本流の世界観を踏襲しすぎず、生活インフラ機能（健康／学習／家計）を一段組み込むと、削除されにくいホーム画面常駐アプリになる。

## 比較で見るべきタイトル

- あつまれ どうぶつの森 — 現実時間と非同期に紐づいた収集ループ。
