## なぜ REW 主分類なのか

プレイヤーの主動機は「著名アーティストの楽曲が紐づいた美少女キャラを引き当て、編成し、眺める」こと、つまり**集める**にある。本作はオートバトル前提の放置系RPGで、操作介入の余地は極小に設計されており、プレイヤーが能動的に「戦う」「読む」体験を取りに行く構造ではない。

concept にある通り、本作の差別化は「楽曲ガチャ」という設計、すなわち**キャラごとに専属の書き下ろし楽曲とイラストが紐づき、ガチャを引くこと自体が音楽・アート消費の儀式になっている**点にある。target 層も「低拘束で美少女キャラを集めたい層」「ボカロ・サブカル系音楽ファン」と明示されており、キャラと楽曲の獲得・所有が動機の中心に据えられていることがわかる。social_axis が null なのは、ギルド戦やリアルタイム PvP が動線の主軸ではなく、ランキング・対戦は補助的にしか効いていないことの裏返しである。

副次的に NAR が効いており、Lament（嘆き）と総称される世界観テキストや、キャラ別の固有ストーリーが「楽曲＋イラスト＋ボイス＋断片的物語」のパッケージとして収集対象の付加価値を上げている。

## 報酬ループの組成

| 層 | 中身 | プレイヤー動機 |
|---|---|---|
| 入手層 | 楽曲ガチャ（キャラ＝楽曲＝イラストが一体） | 推し楽曲・推しアーティストの担当キャラを引く |
| 育成層 | レベル / 装備 / 専用突破素材 | 引いたキャラを「飾れる」状態に持ち上げる |
| 編成層 | パッシブ・属性相性のオート編成 | 放置効率を最大化する最低限のパズル |
| 鑑賞層 | Live2D 演出、フルコーラス楽曲視聴、キャラ別ストーリー | 引いた後の「眺める時間」を伸ばす |
| 競争層 | アリーナ / ボスランキング | 育成成果の確認に留まる軽い競争 |

放置系RPGの「寝ている間に資源が貯まる→ガチャを回す→眺める」ループに、**楽曲という非ゲーム的かつ非消費的な報酬**を組み込んだのが構造的な発明である。アイテムはインフレで陳腐化するが、書き下ろし楽曲は資産として残り、ガチャの長期的な引き動機が劣化しにくい。

## バズ要因 / 人気の根拠

JSON の popularity「著名アーティスト起用の楽曲とイラストで話題化」を分解すると次のようになる。

- ボカロP・サブカルシーンの著名アーティストにキャラ別書き下ろし楽曲を発注し、**音楽コンテンツとしての発信力**をゲーム外（YouTube / 音楽ストリーミング）に拡張した
- 1 周年時点で世界累計 180M ドル超、日本国内 DL 300 万超に到達し、放置系というジャンル制約の中では収益性で上位に入った
- Live2D とイラストの統一感が SNS でのスクリーンショット拡散と相性が良く、キャラ実装ごとに楽曲 PV が独立した話題化単位を作った
- 低拘束プレイで「ながら視聴」「サブゲー」需要を取り、メインの重課金タイトルと共存させやすい設計だった

## 企画への示唆

- ガチャ報酬に**ゲーム外で資産化するコンテンツ（楽曲・MV）**を紐づけると、インフレで陳腐化しにくく長期リテンションが上がる
- 操作負荷を意図的に下げると、コンテンツ消費（ストーリー・音楽・イラスト鑑賞）が主軸の層を取り込める
- 著名クリエイターをキャラ単位で割り当てると、**実装ごとに独立した PR ユニット**ができ、運営期間中の話題化リズムを安定させやすい
- 「世界観の統一テキスト＋キャラ別断片ストーリー」の二層構造にすると、NAR を主軸にしなくても収集対象の意味付けが厚くなる
- social_axis を弱く設計する場合、競争層は「育成確認用の軽量ランキング」に留めることで、放置系の低拘束という価値提案を壊さずに済む
