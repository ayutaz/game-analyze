# 呪術廻戦 ファントムパレード

## なぜ REW 主分類なのか

プレイヤーの主動機は「推しキャラを引き、育て、編成して原作シーンを再演する」ことに集約される。concept にある「キャラガチャを軸」「王道スマホRPG」が示す通り、プレイの中心はガチャ排出→育成→クエスト周回というレアリティ駆動のループであり、戦闘そのものはコマンド RPG というモバイルでもっとも操作介入の薄い形式で実装されている。target が「キャラ収集とストーリー追体験を両立したいライト RPG 層」と明示されている通り、操作スキルの上達や対人競技性は主たる訴求ではない。

ただし secondary に NAR が立つのは、原作補完を謳うオリジナルシナリオの存在が大きい。アニメ調フルボイスの章立てがガチャ更新と連動し、「新章→限定ピックアップ→次の新章」というサイクルでキャラ収集と物語消費を一本化している。social_axis が未設定なのは PvP / 同盟戦のような明確な対他者軸を持たず、ランキングやレイドが副次機能に留まるためで、運用形式としてはソロ＋非同期の中間に位置する。

## 報酬ループの組成

ファンパレの報酬層は「キャラ排出」「育成リソース」「シナリオ解放」の 3 系統が IP 由来の名場面演出で繋がる構造をとる。

| 層 | 主な要素 | リテンション貢献 |
|---|---|---|
| ガチャ排出 | 星 3 排出率と限定ピックアップ、原作章ごとの新キャラ追加 | 新章実装に合わせ課金動機を更新 |
| キャラ育成 | レベル / 凸 / 装備 / 領域展開モーション | 1 体を仕上げる長期目標、デイリー周回の根拠 |
| シナリオ | 原作補完オリジナルストーリー、フルボイス | キャラ実装＝物語進行で停止しにくい |
| イベント | アニメ放映期コラボ章、復刻、ログイン | アニメ放送スケジュールに同期したスパイク |

戦闘システム自体は属性相性とスキル順のコマンド RPG なので、コアの差別化は「誰を編成しているか」と「シナリオ進行で何が起きるか」の 2 点に寄せられている。

## バズ要因 / 人気の根拠

JSON の popularity「アニメ2期と同時期、原作補完シナリオで話題化」を起点に分解する。

- アニメ第 2 期（渋谷事変）放送と配信タイミングを重ね、原作ファンの関心ピークに合わせて新規流入を取り込んだ
- 公式に「原作補完」を打ち出すオリジナルシナリオが、原作既読層にとっても消費価値のある新規コンテンツとして機能した
- 日本国内で 1000 万 DL を突破（2024 年）し、IP モバイル RPG として一定規模の運用ラインに到達
- 開発 Sumzap・配信 Bandai Namco という、白猫プロジェクト系の運用ノウハウと IP ホルダーの広告火力を組み合わせた体制

## 企画への示唆

- IP モバイル RPG をアニメ新章と同期配信すると、初動の認知獲得コストを大幅に圧縮できる
- 戦闘を操作軽量なコマンド式に寄せると、原作ファンの「キャラを触りたい」需要を取りこぼさず広い年齢層を抱え込める
- 「原作補完」を編集方針として明示すると、既読層がコンテンツ消費の正当性を見いだしやすく、ガチャと物語が同じカレンダーに乗る
- 領域展開など IP 固有の演出をガチャ排出時の最大の見せ場に据えると、SR 体験そのものがマーケティング素材として SNS に流出する
- IP 提供元と運用デベロッパを分離する体制は、原作監修と運用判断のトレードオフを設計時に明文化しておかないとシナリオ更新速度がボトルネックになる
