# ピクミン ブルーム

## なぜ REW 主分類なのか

プレイヤーの主動機は「歩いて苗を花に育て、図鑑を埋める」こと、すなわち**集めて記録を積む**ことにある。Niantic の位置情報ゲームでありながら、Pokémon GO と違って対戦も捕獲バトルも存在しない。残るのは「歩数 → 苗 → 花 → 花びら → 花壇・図鑑」という育成と収集の単線ループであり、これは典型的な REW（報酬型）の骨格である。

concept が示すとおり、本作は「健康アプリと収集ゲームの融合」であり、target も「健康・ウォーキング習慣に関心のある成人層」と「対戦が苦手なライト層」に置かれている。つまり競争 EXP を意図的に剥がし、報酬獲得の手触りだけを残した設計だ。secondary に EXP が入るのは、歩くという行為自体・季節イベントで街に花を咲かせる行為が体験として効くため。social_axis が null なのも、明確な PvP / 協力 / 観戦の軸を持たないことを反映している（強いて言えば「ゆるい非同期」）。

## 報酬ループの組成

| 層 | 通貨 / オブジェクト | 取得条件 | 機能 |
|---|---|---|---|
| 入力層 | 歩数 | 端末センサーで自動計測 | 全ループの燃料 |
| 一次産物 | 苗 → ピクミン | 一定歩数で発芽 | コレクション基礎単位 |
| 二次産物 | 花びら | ピクミンが摘んでくる | 街マップを染める消費財 |
| 累積指標 | 花壇 / 図鑑 / 月次レポート | 種類別・期間別に蓄積 | 長期リテンションの錨 |
| 季節層 | 限定デコピクミン・コラボ苗 | 月替わりイベント | リエンゲージ装置 |

前世代の Pokémon GO が「捕獲 → 強化 → レイド対戦」という攻めのループを持つのに対し、Bloom は**入力（歩数）が完全に受動・無意識**で、出力（花を咲かせた街・月次サマリ）が**記録メディア**として機能する点が違う。「今日何歩歩いたか」を可視化する健康アプリの文法を、ピクミン IP の柔らかさで包んでいる。

## バズ要因 / 人気の根拠

- popularity に集約された「ピクミン愛と歩数記録、戦わない位置ゲー」が初動の語られ方を決めた。Pokémon GO 疲れ層の受け皿という文脈で広まった。
- 日本初動 2 週間で **86.4 万 DL**、累計収益 **1 億ドル超**（JSON 記載）。Niantic タイトルとしては GO に次ぐ規模感を保つ。
- 「対戦がない」「他人を倒さない」設計が、ソーシャル疲れ・PvP 苦手層の女性ユーザー獲得につながった。
- 月次レポート機能が SNS で「今月どこ歩いた」と共有されることで、ゲームというよりライフログツールとして可視化された。

## 企画への示唆

- **入力を無意識化すると継続率が跳ねる**：能動的プレイ時間ではなく、歩数という生活ログを入力に置き換えると、起動忘れによる離脱が構造的に減る。
- **PvP を抜くと別のセグメントが開く**：同じ位置情報フレームでも、対戦・捕獲を外すと「健康アプリ」「ライフログ」のユーザー層を取れる。市場を変えずに顧客を変える設計。
- **記録メディア化が長期定着の鍵**：月次レポート・図鑑・花壇のような「振り返り装置」を内蔵すると、報酬ループがアルバム化して捨てづらくなる。
- **IP の柔らかさで生活密着を許容させる**：ピクミンのキャラクター造形が「毎日連れて歩く」抵抗感を消す。シリアス IP では生活密着型は組みにくい。
- **季節 × 場所の限定がリエンゲージを生む**：月替わりイベント・地域限定デコは、能動的に通勤路を変える動機になり、位置情報ゲームの強みを地味に活かす。
