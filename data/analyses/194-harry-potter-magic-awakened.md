# ハリーポッター 魔法の覚醒 — 詳細分析

## なぜ REW 主分類なのか

プレイヤーの主動機は「魔法カードを集めて自分のデッキを編成し、ホグワーツ生としての自分を育てる」こと。すなわち**集める／育てる**が中心にあり、対戦やストーリーはその素材を活かす場として機能する。

concept にある通り本作は「カードバトル RPG」を骨格とし、ガチャと収集を報酬ループの軸に据える。target である「IP 愛着で課金可能なライト〜ミドル層」が継続的に回すのは、**杖・呪文カード・寮・着せ替え・パートナー**といった収集スロットの拡張であり、ストーリー進行は新カード・新章を解放するためのトリガーとして配置されている。学園生活シミュレーションも「友情度を上げて固有カードを獲得する」という収集ループに組み込まれている構造で、純粋な物語消費（NAR）には収まらない。

secondary に NAR を置いたのは、原作 IP の章立てと固有キャラクターの絆イベントが報酬解放の導線として強く効いているため。**IP の物語資産を、ガチャ排出と育成段階の正当化に転用している**設計と読める。

## 報酬ループの組成

カードバトル RPG × 学園シミュレーションのハイブリッドゆえに、報酬層は複線で組まれている。

| 層 | 主な要素 | リテンション役割 |
|---|---|---|
| デイリー | 授業ミニゲーム、寮ポイント、課題 | ログイン動機・スタミナ消費 |
| カード収集 | 呪文・仲間・伝説カードのガチャ | 課金主動線、デッキ多様性 |
| キャラ育成 | カードのレベル / 星 / エコー強化 | ガチャ後の長期消化 |
| 対戦 | 1v1 デュエル、決闘クラブ、ランクマ | 育成成果の試金石 |
| 物語 | メインストーリー章、季節イベント | 新カード / 新衣装の解放トリガー |
| 装飾 | 寮の部屋カスタム、衣装、ペット | IP 没入感と表現欲求 |

特徴は、**カード=戦闘リソースと、寮/衣装=自己表現リソースの 2 系統を並列に走らせている**こと。前者は対戦で消化され、後者は SNS 共有・キャラクター演出で消化される。`social_axis` が null なのは、対戦・友情・ランキングが併存し、どれか一軸とは断言しづらいデザイン上の事情と読める（実態はランキング/同盟戦寄りの非同期色が濃い）。

## バズ要因 / 人気の根拠

popularity の「原作ファン層の取り込みと学園生活演出の評価」を分解すると：

- **IP の生活感を再現**：授業・寮対抗・禁じられた森・ホグズミード散策などの「学園日常」シーンを丁寧に作り込み、原作読者の記憶に刺さる導入を提供。
- **NetEase の運営力**：中国 NetEase 制作で、ガチャ調整・季節イベント・コラボ更新のサイクルが速く、初年度推定で **$500M+** の世界収益を計上。
- **カードゲームとしての遊びごたえ**：呪文カード×仲間カード×エコーの組み合わせで、IP 商法に留まらないデッキ構築の手触りを担保。
- **グローバル同時運営**：iOS / Android に加え PC クライアントを提供、原作の多言語ファンダムを横断的に取り込む設計。

## 企画への示唆

- **IP の「日常」を切り出すと、戦闘以外の課金面が立ち上がる**：寮カスタム・制服・ペット等の自己表現層が、戦闘とは別軸の課金理由になる。
- **物語章をガチャ解放の正当化に使う**と、原作ファンの心理的抵抗が下がる：新章＝新カードという文脈を作ると、ピックアップが「集めるべきもの」に変わる。
- **収集を「カード」と「装飾」の二系統で走らせる**と、戦闘に飽きた層を装飾側に逃がせる：単一系統の収集は飽和すると離脱に直結するが、二系統は相互送客になる。
- **授業ミニゲーム＝デイリーログインの儀式**にすると、原作モチーフがそのまま習慣化装置として機能する：意味づけのあるデイリーは形式的なログボより強い。
- **PC クライアント併売**は、長時間プレイ層と配信文化を取り込みつつスマホ課金導線を維持する両取り戦略になる。
