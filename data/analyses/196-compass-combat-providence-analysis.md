## なぜ EXP 主分類なのか

「#コンパス」を遊ぶ動機は、3 分前後のリアルタイム 3on3 で陣取りを「競う」ことにある。プレイヤーの主動詞は **戦う／競う** であり、ガチャでヒーローや楽曲・カードを揃えるのは「次の試合をどう戦うか」のためのリソース調達である。

concept にもある「片手で遊べる 3on3 陣取り対戦」が体験の中心であり、target も「手軽な対戦を求めつつ、推しヒーロー／コラボ楽曲の収集にも反応する」と、対戦体験を主軸に据えた上で REW を副次に置く構造を示す。`social_axis: 対戦` は文字通り PvP マッチングの上に成立しており、フレンド対戦・ランクマッチ・大会シーンを牽引する基盤となっている。

`secondary: REW` はガチャによるヒーロー／カードの収集と強化、コラボ楽曲・コスチュームの限定排出に表れる。ただしガチャだけ回しても試合に勝てない設計（操作とロール理解が支配的）なので、REW は主動機にならず EXP を支える層に留まる。

## 体験の核 / 設計の発明

| 要素 | 内容 | 前世代 MOBA との差分 |
|---|---|---|
| 試合長 | 1 戦約 3 分 | LoL / DOTA2 の 20〜40 分から大幅短縮 |
| 操作 | 片手スワイプで移動、タップでスキル | キーボード＋マウス前提を放棄 |
| マップ構造 | 3 拠点の陣取り（占拠率勝負） | レーン＋ジャングル＋ネクサスを単純化 |
| デッキ | ヒーロー 1 体＋カード 4 枚 | アイテムビルドをカード事前構築に置き換え |
| IP レイヤ | ボカロ／ニコニコ／アニメコラボ | オリジナル IP 一本足ではない |

設計の発明は **「PC MOBA の 3 分モバイル化」と「カードデッキ構築の事前化」** にある。試合中の判断ポイントは「陣を踏むか、敵を叩くか、味方を守るか」の三択に絞られ、リアルタイム性とロール（アタッカー／タンク／スプリンター／ガンナー）の役割分担で深さを担保している。`social_axis: 対戦` は同期 PvP として、ランクマッチと公式大会（コンパスフェス等）でロングテール化する。

## バズ要因 / 人気の根拠

popularity の「ニコニコ系 IP コラボと 3 分対戦の手軽さ」を分解すると以下になる。

- 国内累計 **25M+ DL** に到達し、日本のモバイル MOBA カテゴリで継続的な首位級ポジションを維持
- ボカロ（初音ミク等）、ニコニコ動画発楽曲、深夜アニメ／少年漫画 IP のコラボヒーローを継続的に追加し、ニコニコ文化圏と地続きのファン層を吸引
- 1 戦 3 分という短さがスマホでの空き時間消化と相性が良く、配信・実況コンテンツの単位としても扱いやすい
- リアルイベント「コンパスフェス」や TV アニメ「#コンパス 2.0」展開で、対戦コミュニティと IP ファン文化を行き来させる導線を維持

## 企画への示唆

- **PC ジャンルをモバイルに移植するときは試合長を 1/5 以下に圧縮する**と、片手・隙間時間のユースケースに乗る
- **操作を 2 入力（移動＋スキル）に絞る**と、MOBA のロール分担・タイミング読み合いを保ったままタッチ UI で成立する
- **デッキ構築を試合外に出す**と、リアルタイム戦闘の判断負荷を下げつつ REW（カード収集）の動機を共存させられる
- **IP コラボをガチャ排出ではなくプレイアブルヒーロー化する**と、コラボ毎にメタが動き、対戦勢と IP ファンの両方を回せる
- **短試合 × 大会フォーマット**を組み合わせると、観戦・配信コンテンツのテンポが整い、コミュニティが二次創作（動画・楽曲）まで広がる
