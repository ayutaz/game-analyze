# Variable Barricade (蜜蜂とイカルス / バリケード盤)

## なぜ NAR 主分類なのか

プレイヤーの主動機は「資産家令嬢ヒバリと 4 人の婿候補＋専属執事の関係性が転がっていく恋愛劇を読み進める」こと、つまり物語消費にある。Otomate の乙女ゲームというジャンル前提自体が「キャラクターごとの個別ルートをコンプリートする」読書型体験を期待する層に向けて作られており、本作の「バリケード盤」も「恋愛／理性」の二軸感情パラメータも、最終的にはどの男性キャラの何という物語を見るかを選ぶための分岐コントローラーとして機能する。

`social_axis` が null なのも妥当で、本作はオンライン要素を持たず、他者との関わりではなく作中キャラとの一対一の関係構築に閉じている。`secondary` が空である通り、収集要素（CG コンプ等）は副次的にあっても主動機を奪うほどには前面化しておらず、あくまで「読み切ったご褒美」の位置に留まる。

## 物語の駆動構造

ロマンチック・コメディを軸にした乙女 VN の標準構造に、ルート探索を可視化する独自レイヤーを 1 枚被せたのが本作の設計。

| 構造要素 | 機能 |
|---|---|
| バリケード盤（盤面状フローチャート） | 攻略順を視覚化し、プレイヤーが自分でルート計画を立てる |
| 「恋愛」「理性」二軸パラメータ | 単一好感度ではなく感情の質を切り分け、同じ選択肢でも別ルートへ振り分ける |
| 4 婿候補＋執事の固定編成 | キャラを絞り、各個別ルートに濃度を与える |
| ロマコメ基調 | シリアス耐性が低い読者でも入りやすく、コメディ落ちで読了感を担保 |

二軸パラメータは「好かれている／嫌われている」を「ときめき度／論破され度」に分解するため、プレイヤー側のメタ的なルート最適化がそのまま物語の駆動エンジンになる。

## バズ要因 / 人気の根拠

- 開発元 Otomate は Hakuoki / Amnesia / Code:Realize で乙女 VN 市場の中核を担うブランドで、その代表作の一つとして 2019 年 PS Vita 末期に投入された
- 2020 年に Switch 移植が出たことで Vita 撤退後の乙女ゲーマー層を取り込み直した
- 2022 年に Aksys Games が北米英語版をリリース、海外乙女ファンコミュニティ（English Otome 系 Wiki / Siliconera レビュー）でロマンチック・コメディの良作として継続的に言及される
- 「バリケード盤」「二軸パラメータ」というシステム面の差別化要素がレビューで取り上げられやすく、純粋な分岐ルート型 VN と並べたときに語る切り口を提供している

## 企画への示唆

- ルート分岐を盤面 UI で可視化すると、プレイヤーは「攻略計画」を立てる楽しみを得て、選択肢単位の意思決定が物語の能動的編集体験に変わる
- 好感度を二軸化すると、「同じキャラに対して 2 種類の関係性を作り分ける」副ルートが自然に発生し、コンテンツ量を実質倍化できる
- キャラクター数を 5 人前後に絞り込むと、個別ルートの密度を上げられ、コンプ目的のリプレイ動機が成立する
- 末期ハード（PS Vita 2019）でリリースしても、次世代機移植＋海外ローカライズの 2 段ロケットを組めば商品寿命を 3 年以上引き延ばせる
- ロマコメ基調を選ぶと、シリアス系乙女 VN との棲み分けが効き、ブランド内のラインナップ補完枠として位置付けられる
