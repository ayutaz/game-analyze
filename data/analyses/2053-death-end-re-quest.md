# デスエンドリクエスト (Death end re;Quest)

## なぜ NAR 主分類なのか

プレイヤーの主動機は「VRMMO に閉じ込められたヒロインを救うために、分岐する複数の Death end ＝ 物語の壊れ方を読み解く」ことにある。動詞で言えば「読む／回収する」が中心で、戦闘は物語介入のための装置として機能する。

concept に明記されている通り、本作は「RPG × VN ハイブリッド」を標榜しつつも、駆動軸は VN 側のマルチエンディング構造に置かれている。VRMMO 側のヒロイン視点と現実側のプログラマー視点が交互に切り替わり、プログラマーが「バグ・選択肢・物語介入」で世界を書き換えるという仕掛けそのものが、プレイヤーに「次のルートで何が壊れるか」を読み進める動機を作る。target も「シナリオ重視で複数エンディングを回収したい層」と明示されており、収集や対戦ではなく物語消費を主眼に置いた設計と一致する。

secondary に EXP が入っているのは、円形アリーナを使う 3 人パーティ制バトルとグリッチ化システムが副次的な体験価値を持つため。ただしバトルそのものを目的化するプレイヤーは想定対象ではなく、あくまで分岐条件・デスエンド誘発のための「操作可能なシナリオ装置」として機能している。

## 物語の駆動構造

本作のシナリオ駆動は「二視点 × 介入可能性 × 分岐エンディング」の三層で組まれている。

| 層 | 仕掛け | プレイヤーに与える駆動力 |
|---|---|---|
| 視点層 | VRMMO 内ヒロイン視点 / 現実側プログラマー視点の切替 | 片方の視点では見えない情報が他方で開示され「もう一周読みたい」動機を生む |
| 介入層 | プログラマーがバグ・コード・選択肢を流し込んで物語を書き換える | 「自分の操作が物語を歪めた」体験を可視化し、分岐への責任感を演出 |
| 帰結層 | Death end（バッドエンド）が条件達成で多数発生するマルチエンディング | 1 周あたりの読了密度を上げ、回収欲をリテンションに転化 |

ダークファンタジー / ホラー寄りのトーンは「Death end」というワードと整合しており、バッドエンドを「失敗」ではなく「収集対象の物語」として再定義した点が VN 文脈での発明にあたる。グリッチ化した世界に踏み込むほどデスエンドが分岐するという設計は、プレイヤーの探索行動とシナリオ枝分かれを直結させている。

## バズ要因 / 人気の根拠

- 「VRMMO 閉じ込め」題材を、SAO 直系の冒険譚ではなくダークホラー × プログラマー視点で再解釈した珍しい組み合わせ
- Compile Heart の Galapagos RPG 第 4 弾という位置付けで、ネプテューヌ系の明るい路線とは異なる「IF/CH のダーク路線ブランド」を確立
- マルチエンディング × Death end の名称設計が SNS で扱いやすく、ルート別の感想・考察コンテンツが二次的に広がりやすい
- PS4 / PC (Steam) / Switch とマルチプラットフォーム展開し、後続の続編・派生（Death end re;Quest 2、Code Z）にシリーズ化したことで認知が累積

## 企画への示唆

- バッドエンドを「収集アイテム」として再定義すると、失敗体験そのものをリテンション資源に変えられる
- 物語内キャラクター（VRMMO 側ヒロイン）と物語外プレイヤー代理（現実側プログラマー）の二視点を用意すると、介入の物語的正当性が自然に立ち上がる
- 「コード／バグ／グリッチ」を物語装置として扱うと、選択肢 UI を単なる分岐選択ではなく「世界改変ツール」に格上げできる
- RPG 戦闘を VN 側のシナリオ分岐条件に紐づけると、戦闘デザインが「報酬獲得」ではなく「物語制御」の意味を持ち、NAR タイトルでも EXP 要素を腐らせない
- ダーク路線をブランド第 N 弾として明確にナンバリングすると、本流タイトルの客層を侵食せずにニッチ層を取りに行ける

## 比較で見るべきタイトル

- シュタインズ・ゲート（マルチエンディング × 観測者視点）
- ダンガンロンパ シリーズ（ダーク VN × 死＝物語イベント化）
- ネプテューヌ シリーズ（同 Compile Heart / IF の本流ブランド比較対象）
- SAO ゲーム化作品群（VRMMO 閉じ込めモチーフの先行例）
- ニーア オートマタ（複数周回でしか全体像が見えない構造）
