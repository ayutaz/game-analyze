## なぜ EXP 主分類なのか

プレイヤーの主動機を 1 文で言えば「友人と地下に潜り、暗闇を切り拓きながら拠点を共同運営する体験そのものを楽しむ」である。Core Keeper は採掘・建築・農業・ボス討伐というサバイバルクラフトの全要素を備えているが、JSON の `concept` が示すとおり中央のコアを起点に最大 8 人で世界を広げる構造であり、`social_axis: 協力` がそのまま遊びの主軸になっている。報酬としてのアイテム獲得や装備強化は確かに濃いものの、それらは「次のバイオームに踏み込む」「より深い闇に光を届ける」体験の燃料として消費される設計で、ガチャ的な収集をゴールに置くタイトルとは構造が異なる。

副次的に効くのが `secondary: REW` で、装備掘り出し・素材ファーム・農作物育成といったループが、協力探索のセッション間を埋める「次に来たときの動機」として機能している。EXP を主軸に置きつつ REW で再訪を作る、いわゆるサバイバルクラフトの定型である。

## 体験の核 / 設計の発明

Core Keeper の発明は「テラリアの 2D 横断探索」「Stardew Valley の生活ループ」「Don't Starve 系の暗闇＝危険」を 1 つの俯瞰視点グリッドに圧縮した点にある。中央のコアという固定起点があるため、拠点が物理的に世界の中心に座り、探索の往復が必ず生活ゾーンを経由する。これにより 8 人 Co-op でも「誰かは農業、誰かはボス前線」といった役割分担が自然に発生する。

| 要素 | 前世代 (テラリア / Stardew) | Core Keeper の差分 |
|---|---|---|
| 視点 | 横スクロール / トップダウン田園 | トップダウン地下、暗闇＝未踏 |
| 拠点 | プレイヤー任意配置 | 中央コア固定で動線が集約 |
| 協力 | 最大 8 人だが目的が拡散 | 8 人が同一拠点を共有、役割分担が起きやすい |
| 進行 | バイオーム巡回 | コアからの放射状探索＋ボス鍵で深層解放 |
| 暗闇 | 演出要素 | 視界＝未開拓そのもの、光源設置が探索行為 |

social_axis の「協力」は、PvP のような明示的対抗ではなく「全員で生活基盤を維持する」型の協力で、ボイチャ前提の重い連携ではなく、隣で別作業をしている軽い同席感が成立するのが設計上の特徴である。

## バズ要因 / 人気の根拠

`popularity` が示す「テラリア×スターデューバレー×地下探索の組み合わせ」が出発点だが、分解すると以下の事実が乗っている。

- 早期アクセス期に既に 200 万本超・プレイヤー数 300 万超を記録し、口コミでサバクラ層に浸透していた
- 2024 年 8 月の 1.0 正式リリースでコンテンツ完結感と「今から始めても遊び切れる」訴求が成立
- 同タイミングのコンソール展開（Switch / PS / Xbox）と Game Pass 収録で接触人口が一段階拡大
- 最大 8 人 Co-op が配信向きで、ストリーマー間の招待・凸が宣伝動線として機能した

数字としては「2024 年で最も遊ばれた Co-op サバクラ」級の指標は持たないが、ジャンル内で Valheim / Terraria / Stardew Valley と並んで言及される位置を獲得した点が定性的な到達点である。

## 企画への示唆

- 既知ジャンル A × B × C の三項合成で打ち出すと、訴求文 1 行で対象プレイヤーが自己同定できる
- 拠点を世界座標の中心に固定すると、Co-op の動線が自然に交差し役割分担が誘発される
- 「暗闇＝未開拓」を視覚的に一意化すると、探索行為そのものが UI 上の進捗表示になる
- 早期アクセスで Co-op を育て、1.0 とコンソール展開とサブスク収録を同時期に重ねると、認知の山を二度作れる
- EXP を主軸にしつつ採掘・農業・装備の REW ループを並走させると、セッション間の再訪動機を確保できる

## 比較で見るべきタイトル

- Terraria（2D サンドボックス、横スクロール採掘の前世代）
- Valheim（協力サバクラの 3D 系、拠点共有モデル）
- Stardew Valley（生活ループ単体の到達点、Co-op を後付けで実装）
- Don't Starve Together（暗闇＝危険、Co-op サバイバルの先行例）
- Necesse（同時期のトップダウン 2D サバクラ、より直接的な競合）
