# アウトラスト・トライアルズ / The Outlast Trials

## なぜ EXP 主分類なのか

プレイヤーの主動機は「仲間と一緒に叫びながら施設から生き延びる」体験そのものにある。前作までの『Outlast』『Outlast 2』はソロのカメラ越し恐怖を一人称で味わうゲームだったが、本作はその IP を 1〜4 人協力に切り替えた時点で「物語を読む」「恐怖の謎を解く」よりも「同行者と恐怖を共有しながら走り切る」プロセスが主目的になった。

concept にある「ソロ恐怖体験をパーティ型恐怖へ転換」と target の「フレンドと絶叫しながらビビり合いたい層」が示すとおり、social_axis は明確に協力。協力ホラーというジャンル指定もこれを裏付ける。secondary に REW が立っているのは、装備・ペリッシュ（投擲アイテム）・スキルツリー・施設内アパートのカスタマイズなど治験の合間に積み上がる育成・収集要素が、リテンションを支える副ループとして機能しているためで、主体はあくまで治験 1 回ごとの体験。

## 体験の核 / 設計の発明

中核は「冷戦期人体実験」というフィクションを使って恐怖をリセットしやすいセッション制に切り出した点にある。Outlast 系の一人称・カメラ・スタミナ管理を維持したまま、1 試行＝1 治験という時間設計に閉じ込めた。

| 設計レイヤ | 前作（Outlast / 2）からの差分 | 協力で生まれる効果 |
|---|---|---|
| 構造 | 単線ストーリーキャンペーン | 治験単位のミッション制／繰り返し前提 |
| 主観 | ソロ一人称・ナイトビジョン | 4 人分の視点が分散、誰かが追われている間に他が前進 |
| 恐怖の強度 | 没入の深さ＝最大化 | 仲間がいる安心と裏切られる瞬間の落差 |
| 失敗コスト | リスポーンで章単位リプレイ | 治験単位で再挑戦、装備とスキルが残る |
| メタ進行 | なし（ストーリー消費型） | アパートのカスタム・装備解放・Rig（特殊能力）育成 |

ソロでも遊べるが、設計上は 3〜4 人時に「役割（救助・索敵・囮）」が自然発生する。配信での悲鳴と笑いの混合は、この役割分担が即興的に生まれることに起因している。

## バズ要因 / 人気の根拠

- 2023 年早期アクセス開始、2024 年 3 月 1.0 ローンチで全機種クロスプレイを開放し、配信者間で同時マッチが容易になった
- 累計プレイヤー数は 2025 年 3 月時点で 300 万人超、累計売上は 200 万本超と公表されており、早期アクセスからの伸びを 1.0 後にもう一段押し上げた
- 既存 Outlast IP のブランド資産（カメラ越し恐怖、Murkoff 社という世界観）を協力に移植したことで、前作配信実況を見てきた層を抵抗なく流入させた
- 同時期に伸びた Phasmophobia / Lethal Company / Content Warning など「配信映えする協力ホラー」潮流の一角として、悲鳴リアクションが切り抜きで拡散しやすい

## 企画への示唆

- 既存ホラー IP をシングルからセッション型協力に移植すると、配信映えと反復プレイの両方を獲得できる（条件：1 試行を 15〜40 分に閉じる）
- 治験という設定で「ステージ＝実験室」と言い切ると、世界観の整合性を犠牲にせず繰り返しミッションを設計できる（条件：フィクション側に反復の理由を持たせる）
- 協力ホラーで役割を強制ロックせず、装備とスキルの組み合わせで自然分担を促すと、4 人時の即興性が上がる（条件：能力差は質的、性能差は小さく）
- メタ進行（アパート・装備・Rig）を治験本体から分離すると、ホラーの緊張と育成の安堵を脳内で切り替えやすくなる（条件：拠点と試合場を明確に分離）
- 早期アクセス→1.0→クロスプレイ解放の三段ロケットは、配信ドリブンの協力ジャンルでは「再注目イベント」を 2 回作れる定石（条件：1.0 で機能追加ではなくアクセス拡大を打つ）

## 比較で見るべきタイトル

- Phasmophobia（同 social_axis: 協力、配信映え協力ホラーの先行例）
- Lethal Company（同 social_axis: 協力、セッション制ホラーの軽量版）
- Dead by Daylight（協力＋非対称、ホラー IP のセッション化）
- Outlast / Outlast 2（同 IP・前世代、ソロ一人称ホラーとの構造差）
- Resident Evil Re:Verse / Resistance（既存ホラー IP の協力／非対称移植の比較例）
