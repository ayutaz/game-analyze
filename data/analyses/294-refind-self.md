# Refind Self: 性格診断ゲーム

## なぜ NAR 主分類なのか

プレイヤーの主動機は「自分という人間像を**読む**」こと。約 1 時間の探索を終えると、その間の選択履歴を疑似データサイエンスが解析し、最終画面で「あなたは何型」という一人称の物語を提示する。ゲームプレイは性格レポートを生成するための入力装置で、出力としてのテキスト（人格像）こそが消費対象になっている。

concept にある「3 周することで全人格パターンを開示する設計」と target の「自分はどんな人間か知りたい層」は、プレイヤーが**自分を主人公にした物語**を回収しに来ていることを示している。social_axis は無印（オフライン単独プレイ）だが、診断結果は SNS に流出して他者と比較される — ここに secondary=EXP の役割があり、操作の自由度（無視するか、話しかけるか、寄り道するか）が「測定行動」として効いている。

## 物語の駆動構造

性格診断という外形を装いつつ、内部は **行動ログ → 解釈テキストの確定的マッピング**で構築されている。ゲーム的に何が起きているかを分解する。

| レイヤー | 仕掛け | プレイヤー体験 |
|---|---|---|
| 探索（約 1 時間） | 移動ルート・NPC への対応・小物への接触を全てログ | 「監視されている」感覚はなく、自由に振る舞っているつもり |
| 終盤の解析演出 | データサイエンティスト風 UI で集計を可視化 | 自分の行動が**意味**を持っていたと事後的に発見する |
| 結果開示 | 16 種前後の人格タイプから 1 つ提示 | 「これが私だ」という解釈テキストを受け取る |
| 周回 | 別の振る舞いで別の人格を引き出せる | 物語の射程が「自分の別人格」へ拡張 |

単線シナリオではなく**解釈エンジンとしてのアドベンチャー**で、章立てや選択肢ノベルとは構造が異なる。プレイヤーは「物語の主人公の決断を選ぶ」のではなく「自分の決断が事後に物語化される」立場に置かれる。

## バズ要因 / 人気の根拠

- 配信開始から 2 周年（2025 年）までで **50 万 DL 超**。インディー診断アドベンチャーとしては異例の拡散規模
- **日本 App Store 無料トップ**に到達。ピクセルアート × 診断という UI 摩擦の低さが効いた
- AUTOMATON 2023 年ベスト選出など国内インディーメディアの年間総括に複数組み込まれた
- 結果画面が**1 画面 1 タイプの自己肯定テキスト**になっており、X（旧 Twitter）でスクリーンショット投稿される前提で設計されている — シェアが宣伝を兼ねる

## 企画への示唆

- **プレイ行動をログに変換し事後解釈する**と、プレイヤーは「自分が主人公」の物語を無料で生成できる
- **完走時間 1 時間 × 3 周設計**にすると、診断系の「もう 1 回試したい」欲求と完走の達成感が両立する
- **結果画面を SNS シェア前提でレイアウト**すると、ユーザー獲得コストを構造的に下げられる（コンテンツ自体が広告素材）
- **疑似データサイエンス UI**で集計を見せると、ランダムな分岐に「測定された」という説得力が乗る
- **ピクセルアート × 非ゲーマー向けジャンル（診断）**を掛けると、コアゲーマー外の流入経路（診断メディア・ライフスタイル系）が開く

## 比較で見るべきタイトル

- 性格診断系 Web コンテンツ（16Personalities など） — UI 摩擦の低さと結果シェア構造の参照元
- Reigns シリーズ — 短時間・カード操作で人格と物語を抽出する設計
- Florence — 1 時間級の短編アドベンチャーで物語を完結させる尺の参照
- Doki Doki Literature Club! — メタ構造で「プレイヤー自身」を物語の対象にする系譜
