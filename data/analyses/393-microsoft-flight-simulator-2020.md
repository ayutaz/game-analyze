# Microsoft Flight Simulator (2020) 分析

## なぜ EXP 主分類なのか

このタイトルの主動機は「実在の地球の上を、実機相当の挙動で**飛ぶ**」という体験そのものにある。勝つこと（REW のランキング）も読むこと（NAR の物語）も中心にはなく、Bing マップと Azure クラウドで再構成された約 2 兆本の樹木・実在地形・実時間気象の上を、コックピットの計器に従って機体を運用する過程自体が報酬になる。

`social_axis: ソロ` が示す通り、設計の中心はマルチプレイ対戦ではなく「世界を眺めながら長時間操縦する」一人称の没入である。`concept` の核は **コンシューマー向けフライトシム再起動**、`target` は既存フライトシム愛好家から、観光・地理・航空に興味のある一般層までを横断する。`secondary: []` で副次的動機は明示されておらず、報酬・物語の上層構造は持たず、純粋な体験設計のみで成立している点が分類を一層明確にする。

## 体験の核 / 設計の発明

中核は「シミュレーション精度」と「地球全体スケール」の同時両立であり、前世代（FSX, 2006）が抱えていた**精度を上げると範囲が狭まる**というトレードオフを、クラウドストリーミングで解消した点に発明がある。

| レイヤー | 担当 | 役割 |
|---|---|---|
| 地形・建物 | Bing Maps + Azure AI による 3D 再構成 | 地球全体を 2PB 級のデータでオンデマンド配信 |
| 気象 | 実時間メテオデータの流し込み | 雲・風・気圧をその瞬間の現実と同期 |
| 機体物理 | 流体力学ベースの空力モデル | 翼面ごとに揚力計算、失速・スピンが固有挙動として現れる |
| インターフェース | コックピット計器の機種別忠実再現 | チェックリスト運用・無線通信が学習対象になる |

`social_axis: ソロ` 環境で、他者ではなく「地球」と「物理」が相手になる構造を取っている。プレイヤーは目的地と機体を選んだ瞬間に、数時間の航路計画と巡航をひとつの長尺な体験として消費する。

## バズ要因 / 人気の根拠

`popularity` の「コンシューマーフライトシムの旗艦再起動／シリーズ 40 年史で最大のローンチ」を分解する。

- ローンチ時 `10M+ players（Game Pass 含む）／1M+ paid` という到達規模で、ニッチジャンルとしては桁違いの初動を出した
- 自宅から実在の街・実家・空港を上空から確認できる「Google Earth 的観光性」が SNS で拡散し、シム未経験層を取り込んだ
- 旅客機・小型機・ヘリ・ジェット等の機体追加が継続配信され、定期的にコミュニティが再点火するサイクルを持つ
- Xbox Series X/S 対応により、PC 専有だったフライトシムをコンソール常設ジャンルに格上げした

## 企画への示唆

- **精度 × 範囲の二項対立を技術スタック側で解くと、新規層を呼び込める**：従来「精緻にすると狭くなる」だったジャンルにクラウド配信を被せると、ニッチがマス寄りに変質する
- **現実データをそのまま素材化すると、観光・郷土性が UGC 動線になる**：自宅・地元・観光地を映す動画が自然発生し、SNS 露出が広告費なしで伸びる
- **ソロ体験でも「時間が長い」設計は配信向き**：長尺フライトは雑談配信・作業 BGM 配信と相性がよく、観戦軸を持たなくても可視性が出る
- **機体追加 DLC のリリースサイクルを定例化すると、ニュース供給がリテンションを支える**：報酬ガチャを使わずに継続課金チャネルを成立させる
- **Game Pass 等のサブスク同梱は、購入障壁の高いシム系で母数獲得に効く**：paid 100 万に対して総プレイヤー 1000 万という比率が、サブスク経由の体験者層を厚くしていることを示す

## 比較で見るべきタイトル

- X-Plane 12（独立系競合、物理エンジン重視）

- FSX（2006 年の前世代、クラウド以前の到達点）
