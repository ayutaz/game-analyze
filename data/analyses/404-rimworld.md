# RimWorld 詳細分析

## なぜ NAR 主分類なのか

プレイヤーの主動機は「自分の入植地に勝手に立ち上がってくる物語を読む（観察する）」ことである。RimWorld はコロニーシムの体裁を取りながら、AI ストーリーテラー（Cassandra Classic / Phoebe Chillax / Randy Random）が確率的にイベントを供給することで、入植者一人ひとりに肉付けされた人格・関係・トラウマが時間軸の上で連鎖していく「物語生成器」として設計されている。開発者 Tynan Sylvester 自身が公式に "story generator" を標榜しており、`genre` の "Colony sim / story generator" 表記もそれを踏襲する。

ビルド／防衛／経済管理という体験ループは存在するが、プレイヤーが事後に語るのは数値最適化ではなく「片腕を失った医者が義娘の死後に酒に逃げ、最終的に襲撃で散った」式の固有名詞付きの逸話である。これは NAR 中核タイトル（ペルソナ／十三機兵）のように「作家が書いた物語を読む」のではなく、「システムが吐く事象列をプレイヤーが事後的に物語化する」型の NAR である。`secondary` は空、`social_axis` も未設定で本作はソロ専用。実質的には EXP（コロニー運営の体験）と REW（DLC / Mod による収集）が下支えしているが、プレイヤーがプレイ後に共有するのは常に「私のコロニーで起きた話」であり、主動機の重心は明確に NAR に寄る。

## 物語の駆動構造

RimWorld の語り口は、AI ストーリーテラーがイベント密度と難度を調整する「ペーシング層」と、入植者の属性・関係・記憶が反応として残る「人物層」の二段構えで動く。

| 層 | 構成要素 | 物語への寄与 |
|---|---|---|
| ペーシング層 | ストーリーテラー 3 種 × 難度 / 襲撃ポイント計算 | 何が・いつ・どの強さで起きるかを確率制御。緊張と弛緩のリズムを与える |
| 人物層 | Backstory / Traits / Skills / Mood / 関係（恋愛・血縁・敵対）/ Memory | 同じ事象でも反応が個別化。「殺人現場目撃」「友人の死」が長期的に行動を歪める |
| 環境層 | バイオーム / 気候 / 病気 / 5 本の DLC（Royalty / Ideology / Biotech / Anomaly / Odyssey） | 物語の舞台設定を変える。各 DLC が新しい語り口を追加 |
| 事後解釈層 | 死亡ログ / 関係ツリー / Mod の物語化 UI | プレイヤーが「これは物語だ」と意味づける手がかり |

中心の発明は **「キャラクター属性が反応として残る」** こと。Mood システムは事象を即時の幸福度に変換するだけでなく、Memory として数日〜永久に持ち越され、無関係なはずの行動（手抜き、突発的犯行、メンタルブレイク）を引き起こす。結果、プレイヤーは「この行動はあの事件への反応だ」と因果を物語化せざるを得なくなる。さらに 5 本の有料 DLC（Royalty 2020 / Ideology 2021 / Biotech 2022 / Anomaly 2024 / Odyssey 2025）が、それぞれ封建制・思想・遺伝子・怪異・宇宙という別ジャンルの物語フックを後乗せし、本体システムの上に新しい語り口を積層させている。

## バズ要因 / 人気の根拠

- Steam 所有者は Gamalytic 推定で約 8.7M（5–10M レンジ）。インディーコロニーシムでは突出した規模で、Dwarf Fortress 系譜のニッチを大衆化した。
- "Most-modded base-builder on Steam" と評されるほど Mod エコシステムが厚く、Combat Extended / Hospitality / Vanilla Expanded 群など大型 Mod が「公式 DLC 相当」の物語拡張として機能している。
- 2018 年正式リリースから 5 本の有料 DLC（最新 Odyssey 2025）を 5 年以上維持。長期サポート型インディーとして PC ゲーマー文化に定着し、DAU と価格が落ちにくい。
- 配信・YouTube・Reddit・日本語 note で「私のコロニーで起きた事件譚」が二次再生産され、配信者の長編シリーズが定番化。同じプレイ動画が二度と発生しないことが視聴コンテンツ化の燃料になっている。

## 企画への示唆

- **属性 × 記憶 × 反応の三角形を確率的に揺らすと、システムは勝手に固有名詞付きの物語を吐き出す。** スクリプト物語を書くより安価で、リプレイ性が桁違いに伸びる。
- **「ストーリーテラー」という難度層を切り出してプレイヤーに選ばせると、ペーシングが体験設計の前面に出る。** 難度バー一本より、世界観に紐づけた語り手の名前で選ばせるほうがプレイヤーは性格を内面化する。
- **ベース体験を変えずに DLC で別ジャンルの物語フックを後乗せできるのは、人物層がジャンル非依存に作られているから。** 中世・思想・遺伝・怪異・宇宙が同じ Mood / Memory システムで統一処理されている。
- **Mod API を物語拡張に開くと、コミュニティが二次的な NAR 装置を作る。** Mod 同士の干渉を許容する設計思想（壊れることを受け入れる）が前提で、運営コストなしで物語多様性が伸びる。
- **失敗エンドを排除しないと、語りの素材が増える。** 全滅すら「語るに足るエピソード」として消費されるよう、勝利条件を緩く・敗北を演出として位置づけると配信文化に乗りやすい。

## 比較で見るべきタイトル

- Dwarf Fortress（物語生成器の原典、UI を犠牲にして深度を取る型）
- Crusader Kings III（人物層を歴史スケールに引き上げた NAR ストラテジー）
- Project Zomboid（ソロ生存 × 事象生成、語り口は終末側）
- Wildermyth（手続き的に英雄譚を生成する別解）
- Stellaris（プロシージャル銀河史 × AI イベントの宇宙版）
