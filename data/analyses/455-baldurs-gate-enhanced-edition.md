## なぜ EXP 主分類なのか

プレイヤーの主動機は「AD&D 2nd Edition のルールを舞台に、6 人パーティを編成して Sword Coast を探索・戦闘する」体験そのものにある。1998 年の Infinity Engine 版を Beamdog が 2012 年に再構築したこの版は、物語の追体験よりも「ルールを学び、ビルドを組み、リアルタイム＋ポーズ戦闘を組み立てる」過程に主な時間が割かれる。social_axis はソロ（旧版はマルチもあったが現代の主動線はシングル）であり、他者との関わりではなく、AI コンパニオン 5 名との関係を含めた「自分で組み上げるパーティ」が体験の核を形成する。

secondary に NAR が入る理由は、Forgotten Realms 世界観と Bhaal の血を継ぐ主人公の運命が長尺の縦糸として機能しているため。ただし物語は分岐や演出の力で消費させるタイプではなく、「探索の報酬」として現れる構造で、駆動力としては副次的。

## 体験の核 / 設計の発明

Enhanced Edition が再提示した中核メカニクスは以下の通り。

| 設計要素 | 効果 |
|---|---|
| AD&D 2e ルールの忠実再現 | THAC0・セービングスロー等のクラシックな計算を学ぶ過程自体がゲーム化 |
| リアルタイム＋アクティブポーズ | アクションの即時性とターン制の思考時間を両立させ、6 人同時操作を可能に |
| 等角投影マップでの広域探索 | 1 マップに複数イベント・敵・隠し要素を詰め、徒歩で踏破する密度感 |
| コンパニオン同士の confict / banter | NPC 同士が反目すると離脱するため、編成が物語を兼ねる |
| EE 追加要素（Dorn, Neera, Rasaad, Baeloth, Black Pits アリーナ） | 旧版未収録の新キャラ・新ダンジョンを差し込み、既プレイ層にも探索動機を再付与 |

前世代（PC-98 系 JRPG や Ultima 系）との差分は、(1) 戦闘の最小単位がキャラ個別行動ではなくパーティ全体の陣形管理であること、(2) ダイスロール結果をログで全公開し、プレイヤーに「ルールを読む楽しさ」を委ねたこと。ソロ軸ながら、6 人を「人間関係を持つ AI」として扱うことで内的な社会性が生まれている。

## バズ要因 / 人気の根拠

popularity 文の起点は「BG3 を可能にした 1998 年 Infinity Engine クラシックの近代化再リリース」。これを分解すると：

- Steam の owner 推定は約 70 万〜100 万、レビュー 1.5 万件超で 92% positive と、20 年以上経った CRPG としては異例の継続評価
- 2023 年の Baldur's Gate 3（Larian）大ヒット後、ナンバリングの起点を遡るプレイヤーが EE 版を経由する流入経路が確立
- iOS / Android / Switch / PS4 / Xbox One への移植で、PC マウス前提だった旧 UI を携帯デバイスで触れるようになった
- Beamdog は同エンジンで BG2 EE、Icewind Dale EE、Planescape: Torment EE を同シリーズ化し、Infinity Engine 作品群の保存・販売を一手に担う構造を作った

## 企画への示唆

- 旧作の UI を現代化するなら「ルール本体は触らず、操作層だけを差し替える」と、原典ファンと新規層の両方を取り込める
- 追加コンパニオンを 3〜4 体差し込むと、既プレイ層に「もう一周する理由」を低コストで供給できる
- 携帯デバイス移植で寿命を延ばす場合、視点（等角投影）はそのままにタッチ UI だけ作り直すと開発コストを抑えられる
- 続編が大ヒットした時に「同エンジンの旧作群」をパッケージ化しておくと、続編からの遡及流入を全部受け止められる
- ダイス結果や内部計算をログで全公開すると、プレイヤーは「ルールを学ぶ遊び」を自発的に始める
