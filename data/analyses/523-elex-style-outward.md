# アウトワード (Outward) — 分析

## なぜ EXP 主分類なのか

プレイヤーの主動機は「**サバイバルの摩擦を受け入れて旅をすること**」である。戦う・読む・集めるのいずれでもなく、近い動詞は「**耐える / 旅する**」になる。Outward の設計は意図的にパワーファンタジーを拒否しており、ザコ相手にも普通に負ける、負ければ装備とリソースを失い、世界はプレイヤー不在のまま進行する。この「自分が中心ではない世界で生き延びる体験」自体が遊びの中核なので、物語消費（NAR）や装備収集（REW）ではなく EXP が主分類となる。

`social_axis` は **協力**。本作は分割画面と オンライン両対応の co-op を備え、二人プレイ時もダメージ・空腹・体温・荷物制限がそのまま二人分発生する。つまり co-op は「強くなるための救済」ではなく「**二人で同じ不便を分担する**」体験であり、協力軸が EXP の核に深く埋め込まれている。`secondary` の NAR は 4 派閥の選択肢を持つメインクエスト、REW はスキルツリーとレシピ収集が担うが、いずれも「旅の途上で起きること」の従属物として配置されている。

## 体験の核 / 設計の発明

Outward のメカニクスを分解すると、AAA オープンワールド RPG が省略してきた**摩擦の層**を意図的に再装着していることがわかる。

| 設計レイヤ | 一般的 RPG | Outward の処理 |
|---|---|---|
| 死亡時の扱い | リスポーン or リロード | 装備を失い、敵 NPC や奴隷商人に拾われ、別の場所で目覚める |
| ファストトラベル | ほぼ無制限 | 原則なし。徒歩 + 食料・水・テント管理 |
| 体力回復 | 自動 / ポーション豊富 | 自然回復なし。睡眠・食事・包帯で段階的回復 |
| インベントリ | 重量制限のみ | 重量 + 個数 + 銀行に預けないと持ち越し不可 |
| 主人公の格 | 選ばれし者 | ただの債務者。家のローンを返すところから始まる |
| co-op の負荷 | 二人で楽になる | 二人分の空腹・体温・敵スポーンが発生 |

前世代（Skyrim / Witcher 3）からの差分は「強くなる曲線をフラットにし、移動と補給を遊びの主要動詞に格上げした」点にある。協力プレイは「片方が死んでももう片方が拾いに来れる」というセーフティを生むため、ソロでの過酷さが**人間関係のドラマに変換される**設計になっている。

## バズ要因 / 人気の根拠

- パワーファンタジーを拒否した hardcore survival CRPG という立ち位置が、ELEX / Gothic / Kingdom Come 系の「不便を肯定する RPG」コミュニティで強い支持を得た
- 分割画面 co-op に対応した 3D オープンワールド RPG が希少で、ローカル協力プレイ目的のカップル / 兄弟層に刺さった
- Steam owners 推定 50 万〜100 万本規模で AAA ではないが、Nine Dots Studio という小規模スタジオの規模を考えれば商業的に十分成立
- 2022 年の続編発表（Outward 2）と Definitive Edition の追加配信により、初動より後年のロングテール売上に寄与した

## 企画への示唆

- **死亡をペナルティではなくイベントに変換すると、世界の能動性が立ち上がる** — リロードを禁じて「敗北後の物語」を分岐させると、敗北自体がコンテンツになる
- **ファストトラベルを削ると、地図が遊びの主役になる** — 補給・天候・体温を移動コストに混ぜることで、徒歩 30 分が退屈ではなく緊張になる
- **co-op を「楽になる装置」ではなく「負荷の共有」にすると、関係性ドラマが生まれる** — 二人とも空腹・寒さ・重量を抱えると、役割分担と裏切りの余地が同時に出てくる
- **主人公を「選ばれし者」から「債務者」に下げると、序盤の動機設計が一気に楽になる** — 救世主任務ではなく家のローンを返す目的なら、プレイヤーは自然に金銭・装備・派閥を選び始める
- **ニッチを狙うときは AAA が省略した摩擦を再装着する** — 不便さを「劣化」ではなく「設計意図」として打ち出せれば、500K-1M の中規模 IP は十分成立する

## 比較で見るべきタイトル

- ELEX / ELEX II (Piranha Bytes) — 同じく「不便を肯定する」ドイツ系オープンワールド RPG
- Gothic シリーズ — 主人公を弱者から始める原典
- Kingdom Come: Deliverance — 現実的サバイバル + 中世 RPG の代表例
- Valheim — 協力サバイバル × 探索のヒット例、co-op を「負荷共有」型に振った設計
- Mount & Blade: Bannerlord — 派閥・経済・敗北後継続のオープンワールド構造
