# Game Dev Tycoon (Greenheart Games, 2013)

## なぜ EXP 主分類なのか

このタイトルの主動機は「自分のゲーム会社を経営して、ジャンル × プラットフォームの最適解を探り当てる体験」であり、プレイヤーが楽しむのは数値の蓄積ではなく**意思決定の試行錯誤**そのものである。`social_axis: ソロ` が示す通り、対戦も協力もない一人称の意思決定ループだが、対象は「自分自身の前回プレイ」であり、各サイクルで前作より良いゲームを出そうとする内的な競争が走る。

`genre: Studio management tycoon` が分類軸として明確で、収集要素（スタッフ、ジャンル解禁、トピック）はあるものの、それらは報酬として消費するのではなく**次の意思決定の選択肢を広げる材料**として効く。したがって REW ではなく EXP（ソロ）が妥当。secondary が空なのは、物語駆動も収集駆動も補助的に効いておらず、シミュレーション体験単体で成立しているため。

## 体験の核 / 設計の発明

中核メカニクスはタイトル開発の 3 軸最適化と、リリース後のレビュー点 → 売上 → 次作予算という長期フィードバックである。前世代の Game Dev Story（Kairosoft, 2010）からの差分を分解する。

| 要素 | Game Dev Story (2010) | Game Dev Tycoon (2013) |
|---|---|---|
| プラットフォーム | iOS/Android 中心、短時間プレイ | PC/Mac、1 プレイ 4-6 時間の長尺 |
| 意思決定の粒度 | ジャンル × トピックの 2 軸 | ジャンル × トピック × プラットフォーム × 開発配分スライダー |
| 失敗の可視化 | 売上数字のみ | レビュー点で要因が逆算しやすい |
| メタネタ | 控えめ | 業界実機の風刺（Vena Oasis = Sega Saturn 等）が前面 |
| DRM 対策 | — | 海賊版コピーには「自社ゲームが海賊版にやられて倒産する」演出 |

ソロ軸ながら、リリース版がメディアで取り上げられたのは最後の DRM ジョーク（海賊版プレイヤー向けに**わざと海賊版を仕込んだ別ビルドを配布**し、ゲーム内で破綻させる仕掛け）であり、ソロ体験の中に「外部のコミュニティ話題」を埋め込んだ点が設計上のフックになっている。

## バズ要因 / 人気の根拠

popularity は「Indie tycoon hit that anchored the modern 'make your own [industry]' sim wave」。これを分解すると：

- Steam 所有者 2M 超、モバイル/PC 合算で約 5M に到達（`sales_world` ベース）し、2013 年時点のインディー Tycoon としては最上位クラス
- リリース直後の「海賊版 DRM ジョーク」がメディアと SNS で広く拡散し、Greenheart Games のブログ記事が世界中のゲーム媒体に転載された
- 「ジャンル × プラットフォーム × 時代」のパロディ表現が、業界に詳しいプレイヤーの考察コンテンツ・配信ネタを誘発
- Software Inc.、Mad Games Tycoon、PC Building Simulator、Computer Tycoon 等の後発 [industry] tycoon 群の「参照点」として引用され続け、ジャンル復興の起点として記憶された

## 企画への示唆

- **「自分の前作」を仮想敵にすると、ソロでも競争のドラマが立つ** — マッチメイクなしでもレビュー点という単一スコアが PvP の代替になる
- **業界そのものをパロディの題材にすると、プレイヤーが二次創作・考察を勝手に量産する** — ネタの自走力がマーケ予算の代替になる
- **DRM 対策をプロダクト演出に組み込むと、ネガティブ施策がポジティブ話題に反転する** — 倫理的な摩擦が逆に拡散の燃料になる
- **意思決定のスライダーは 3-4 本まで** — 多軸最適化の楽しさは「全部触る価値があるが、全部触ると破綻する」帯域で成立する
- **1 プレイ 4-6 時間の長尺ソロ Tycoon は、年代 × プラットフォームの進化を内蔵すると再プレイが伸びる** — 時代変化が「次の周回をやる理由」になる

## 比較で見るべきタイトル

- **Game Dev Story / ゲーム発展国++**（Kairosoft, 2010）— 直接の前世代、モバイル尺の最小実装
- **Software Inc.**（Coredumping, 2015）— Game Dev Tycoon の長尺・深部最適化版
- **Two Point Hospital**（Two Point Studios, 2018）— 業界パロディ × Tycoon の継承例
- **Mad Games Tycoon 2**（Eggcode, 2021）— ジャンル × プラットフォーム最適化の更なる粒度細分化
- **PC Building Simulator**（Claudiu Kiss, 2018）— 「自分の[industry]を作る」系の隣接ジャンル
