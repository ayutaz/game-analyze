## なぜ EXP 主分類なのか

Opus Magnum をプレイする主動機は「自分の手で錬金術回路を設計し、最適化する体験」を味わうことに尽きる。物語要素（19世紀風アルケミー帝国の家族劇）は存在するが、それは題材であって駆動装置ではない。プレイヤーは盤面上にアームとトラックを配置し、原子を結合・移動させてゴール分子を組み立てるプログラムを「書く」。動詞は **創る／組む／削る** であり、ストーリーを読み進めるためでも報酬を集めるためでもなく、自分の解法をコンマ秒・1サイクル・1コスト単位で削っていく快感が中核にある。

`social_axis: ソロ` は表層的な属性に見えて、実は本作の体験設計の中心軸である。GIF 書き出しと Steam ランキングで「最終的に他者と並ぶ」構造になっており、孤独な最適化作業の出力が比較可能な数値（cost / cycles / area）として可視化される。secondary が空であることが象徴的で、収集要素も物語消費要素も付随させず、設計と最適化の純度を上げている。

## 体験の核 / 設計の発明

Zachtronics 系の系譜（SpaceChem / TIS-100 / Shenzhen I/O）の中で Opus Magnum が達成したのは「プログラミングパズルの可視化」である。前作までのテキストエディタ上でアセンブリ的命令を書く形式から、平面上のアーム・トラック・命令テープという**3 次元の振り付け**に置換した。

| 設計要素 | 機能 | 前世代との差分 |
|---|---|---|
| 命令テープ（アーム1本ごとに時系列） | 動作のループを記述 | TIS-100 のテキストコードを 2D タイムラインに置換 |
| 評価軸 3 種（cost / cycles / area） | 解の「形」を多次元評価 | 単一スコアでなくトレードオフを強制 |
| GIF 自動出力 | 解法を SNS にそのまま放流可能 | TIS-100 にはなかったバズ装置 |
| ヒストグラム | コミュニティ全体の分布で自分の位置を可視化 | 順位ではなく分布表示で再挑戦欲を刺激 |

特に評価軸が 3 種ある点は重要で、cost を削るとサイクルが伸び、サイクルを削ると面積が膨らむというトレードオフが生まれる。1 つのパズルを「3 回別解で解く」動機が構造的に内包されている。

## バズ要因 / 人気の根拠

- Zachtronics の系譜中で **最もアクセシブル**と評され、テキストプログラミング経験のないプレイヤーにも入口が開かれた
- 視覚的に優雅な錬金術モチーフ × アーム動作 GIF が SNS 拡散に最適化されており、解法を見せる文化が育った
- Steam で 5,000+ レビュー × 評価ほぼ 10/10 という、ニッチジャンルでは極めて高密度の支持
- 推定 ~$2M の収益はインディーパズルとして十分な水準で、Zachtronics の代表作と位置づけられた

## 企画への示唆

- 評価軸を 3 軸用意してトレードオフを作ると、1 パズルあたりの再挑戦回数が倍々に増える
- 解の「過程」を GIF / 動画で書き出せるようにすると、コミュニティが勝手にショーケースを作る
- ランキング表示ではなく**ヒストグラム表示**にすると、上位への到達感ではなく「平均との差」への執着が生まれ、中位層が脱落しにくくなる
- ソロゲームでも数値を公開比較可能にすれば、social_axis=ソロのまま擬似的な対戦体験を作れる
- テキストコーディングを 2D タイムラインに視覚化すると、専門知識の参入障壁が一段下がる

## 比較で見るべきタイトル

- SpaceChem / TIS-100 / Shenzhen I/O（Zachtronics 自身の系譜、抽象度の段階的低下）
- Factorio（最適化欲求の集団化、社会軸が協力に振れた版）
- Baba Is You（ルール改変系パズル、解法の自由度で並ぶ）
- The Witness（ソロ × 視覚的優雅さ × 多軸評価の前例）
