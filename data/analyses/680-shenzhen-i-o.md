# SHENZHEN I/O

## なぜ EXP 主分類なのか

プレイヤーの主動機は「与えられた仕様書の通りに動くマイクロコントローラ回路を、限られた命令数とピン数の中で組み上げる」という、設計と最適化の体験そのものにある。ストーリーは深圳に渡った技術者のメール往復という最小限の枠で、報酬は次の発注書と新しい命令セットしか出てこない。物語の続きを読みたいから進めるのでも、レアパーツを集めたいから進めるのでもなく、「いま自分が書いた回路がもう一段短くなる気がする」という解の改善欲求でセッションが回る。

`social_axis: ソロ` は、Zachtronics 系列の他作（TIS-100、Opus Magnum、Exapunks）と共通する設計思想を反映している。クリア後にステージごとに表示される「サイクル数 / ライン数 / コスト」のヒストグラムは Steam フレンドの分布と並ぶ非同期の腕比べになっているが、対戦やマッチングは存在せず、関わり方の中心はあくまで自分のソルバとの内省的な対話だ。`secondary` が空であることは、収集要素や物語消費が機能していないことの裏返しでもある。

## 体験の核 / 設計の発明

中核は「実在の電子工作カタログを模した仮想 MCU セット」と「アセンブリ風の DSL」の組み合わせにある。プレイヤーは XBus / SimpleIO のピン制約、6 行前後の命令枠、テスト入力の固定パターンを与えられ、回路図とコードを同時に書く。Zachtronics の前世代作との差分は以下のように整理できる。

| 要素 | TIS-100 (2015) | SHENZHEN I/O (2016) | 差分の効果 |
|---|---|---|---|
| 表現対象 | 抽象的なノード並列計算 | 実在風 MCU + アナログ I/O | 仕様書の読解が体験の一部になる |
| ドキュメント | ゲーム内 PDF | 印刷を推奨する詳細マニュアル | 紙でメモを取る儀式が成立 |
| 評価軸 | サイクル / ノード / 命令 | サイクル / コスト / ライン | 部品選定の経済性が加わる |
| 副次モード | なし | ソリティア (深圳上海麻雀) | 詰まったときの息抜きを内蔵 |

「マニュアルを印刷して読み込む」「ピン番号を紙にメモする」という現実世界の作業に体験を染み出させたことが、同社作中でも特に「電子工作の追体験」という固有のジャンル感を作っている。

## バズ要因 / 人気の根拠

- Steam owners 20 万〜50 万本規模、推定総収益 190 万ドル前後と、ニッチパズルとして十分な経済規模を確保。
- 紙の冊子を意識した PDF マニュアルがコミュニティで話題化し、レビューでも「マニュアルがゲームの一部」と語られ続けている。
- Zachtronics シリーズの「エンジニアリングパズル」系譜（SpaceChem → TIS-100 → SHENZHEN I/O → Opus Magnum → Exapunks）の中で、最も実在の組み込み開発の文法に寄せた一作という位置づけ。
- 配信・実況では解法ヒストグラムの上位を狙う最適化挑戦が定番化し、コードを公開して比較する文化が定着した。

## 企画への示唆

- 評価軸を 3 本（速度 / コスト / 行数）に分けてヒストグラム可視化すると、クリア後も同じ問題を解き直す動機が継続する。
- 物語をメール往復だけに削っても、ゲーム内の「発注書」を物語装置として使えばモチベーション維持には十分機能する。
- 仕様書を意図的に紙メディア的にデザインすると、プレイヤーが机を整える儀式ごと体験化し、滞在時間が伸びる。
- 制約（命令数・ピン数・予算）を緩めずに小問を増やす設計は、ステージ単位で挫折してもメタゲームとしての継続率を保てる。
- 詰まった時の逃げ場（ミニゲーム / ソリティア）をクライアント内に内蔵すると、離脱がアプリ外に流れるのを防げる。

## 比較で見るべきタイトル

- TIS-100 — 抽象アセンブリ版の前世代、評価軸とヒストグラム設計の原型。
- Opus Magmum — 同社の空間最適化パズル、評価軸を錬金術の儀式表現に置換した派生。
- Exapunks — 同社作でストーリーと zine 風マニュアルを強化した後継。
- Factorio — ソロ最適化欲求を工場規模に拡張した別系統の代表例。
