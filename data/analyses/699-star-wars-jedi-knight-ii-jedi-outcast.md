# スター・ウォーズ ジェダイナイト II ジェダイアウトキャスト

## なぜ EXP 主分類なのか

プレイヤーの主動機は「ライトセーバーを振る・捌く・斬り抜ける」という剣戟操作そのものを味わうことにある。スター・ウォーズという IP の上に乗ってはいるが、Raven Software が 2002 年に提示した遊びの中核は、3 種類のスタンス（Fast / Medium / Strong）と方向入力で攻撃モーションを切り替え、敵セーバー戦士と斬り合うアクション体験だった。

social_axis が「ソロ」なのは、本編キャンペーンが Kyle Katarn を主人公にしたシングルプレイ FPS / 三人称アクションのハイブリッドで、対戦モードは存在したものの主動機ではないため。secondary に NAR と REW が付くのは、スター・ウォーズ正史脇のストーリーを追う物語的牽引と、フォースパワー（プッシュ・プル・ライトニング等）を章進行で解放していく段階的能力獲得が遊びを支えているからで、いずれも剣戟体験を支える副軸として機能している。

## 体験の核 / 設計の発明

中核メカニクスはライトセーバー戦闘の「双方向な刃の物理」にある。攻撃と防御が同じ刀身で行われ、敵の振り下ろしを自動的に弾く防御挙動と、こちらの振りに方向性を持たせるスタンス切り替えが噛み合うことで、FPS の射撃テンポとは別系統の間合いゲームが成立した。

| 要素 | 設計 | 体験への寄与 |
|---|---|---|
| スタンス制 | Fast / Medium / Strong の三段切替 | 速度と威力のトレードオフ判断が常時発生 |
| フォースパワー | プッシュ / プル / グリップ / ライトニング | 剣戟の外側に状況操作レイヤーを追加 |
| 弾き返し | ブラスター射撃をセーバーで反射 | 銃と剣の交差点を可視化 |
| 章進行 | フォース能力を段階解放 | スキル獲得 = ステージ難度の鍵 |
| カメラ | FPS とサードパーソンを切替可能 | 剣戟は TPS、銃撃は FPS で最適化 |

前世代の『Jedi Knight: Dark Forces II』(1997) からの差分は、Quake III エンジン採用による近接の応答性向上と、セーバーの当たり判定をモーション主導に置き換えたこと。これが後年の『Jedi Academy』(2003) や『Jedi: Fallen Order / Survivor』系の剣戟設計に参照され続けている。

## バズ要因 / 人気の根拠

popularity の「modern Jedi: Survivor チームが今も参照する剣戟の金字塔」を分解する。

- Steam 版の所有者数は 50 万〜100 万本規模、Aspyr によるリマスター移植（Switch / PS4）は 20 万本超。ローンチ規模より「20 年以上売れ続けている」ことが特徴。
- 発売当時の PC 専門誌で剣戟挙動が高評価を受け、MOD コミュニティ（OJP、Movie Battles II の前身）が長期に渡って活動を継続。
- Disney / LucasArts / Aspyr とパブリッシャーが移行する中、IP 権利が変わっても新世代機への移植が続いた数少ない旧 LucasArts タイトル。
- 公式に「ライトセーバー戦闘の参照点」として後発作のスタッフが言及することで、ゲーム外メディアでも引用されるレガシー資産になった。

## 企画への示唆

- 単一武器の挙動を「攻撃・防御・反射」の三役で兼ねさせると、入力対効果の密度が上がり剣戟一本で 1 タイトル成立する。
- スタンス切替のような少数モード切替を入れると、アクション入力の語彙を増やさずに戦況判断レイヤーを足せる。
- 主武器（セーバー）の外側に状況操作系スキル（フォース）を置くと、戦闘設計が「剣戟 × 環境操作」の組み合わせ問題に拡張される。
- 章進行で能力解放を結びつけると、ステージ設計とプレイヤー成長を同じ目盛で語れる（NAR・REW を副軸に取り込みやすい）。
- 強い IP に乗る場合でも、IP 固有要素（セーバー、フォース）をそのままコアメカニクスに翻訳できれば、ファン需要と独立した遊びの軸が立つ。

## 比較で見るべきタイトル

- Jedi Academy (2003) — 直系後継、スタンスとセーバー組み立てを拡張
- Star Wars Jedi: Fallen Order / Survivor — 現代版のセーバー剣戟、参照元として明示
- Dark Forces II: Jedi Knight (1997) — 前世代、FPS とフォースの原型
- SEKIRO: SHADOWS DIE TWICE — 弾き / 体幹で双方向な刀の物理を別解で実装した近接アクション
