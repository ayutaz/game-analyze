# Astrea: Six-Sided Oracles

## なぜ EXP 主分類なのか

プレイヤーの主動機は「ダイスの目を構築し、確率を曲げ、戦闘の手触りを試す」こと。`concept` が示すとおり、本作はカード DBG の語彙を「ダイス」に置換した戦術ローグライトであり、`target` も「ダイス／確率マネジメントの戦術的な手触りを好む層」と明示している。プレイヤーが毎ラン何度も触りに戻る理由は、Oracle ごとの構築解を試す**ビルド体験そのもの**にあり、ストーリー消費でもガチャ的収集でもない。

`social_axis: ソロ` のシングル完結ローグライトであり、`secondary: REW` が示すのは「ラン内でダイス面を強化／変質させていく構築の成長カーブ」が報酬として効いている点。ただし蓄積される報酬（永続アンロック・コスメ）よりも、ラン内で偶発的に組み上がるシナジーを味わうことが主動機のため、REW は副次にとどまる。

## 体験の核 / 設計の発明

中核は二軸ダメージと六面ダイスの「面交換」構築。Slay the Spire 以降の DBG が「カード=単発リソース」を組み合わせるのに対し、本作は「6 面のうち何面を Purification にし、何面を Corruption にし、何面を空白に潰すか」というダイス単位の編集を構築操作に据えた。

| 設計レイヤー | DBG 系の通例 | Astrea の置換 |
|---|---|---|
| 構築単位 | カード 1 枚 | ダイス 1 個（6 面） |
| 構築操作 | 追加 / 削除 / アップグレード | 面の追加 / 削除 / 置換 / 空白化 |
| ダメージ軸 | HP 単軸 | Purification（味方回復／敵ダメージ）と Corruption（味方汚染／敵回復反転）の二軸 |
| 偶発性の源 | ドロー運 | 出目運 ＋ ドロー運 |
| 失敗状態 | HP 0 | Corruption 値が閾値超過 |

Purification と Corruption が「敵味方どちらにも作用しうる」ため、自分を汚染する目を含むダイスを敢えて採用してリスクを取りに行く構築が成立する。出目の確率は「面数 / 6」と明示的に可視化されており、運の塊ではなく**期待値を編集するゲーム**になっている点が前世代との差分。`social_axis: ソロ` のため、他者比較ではなく自分の試行回数だけで価値が立ち上がる構造。

## バズ要因 / 人気の根拠

- `popularity` が示すとおり、Slay the Spire 系の飽和ジャンルに「dicebuilder」という独自切り口を持ち込み、海外メディア（PC Gamer 等）の好評で初動の認知を獲得した。
- 2023 年 PC リリース → 2024 年コンソール展開 → 2025 年モバイル展開と、**プラットフォーム階段拡張**で長期に再露出を稼いだ。
- Oracle ごとに異なる初期ダイス構成・固有メカニクスを持つキャラクター制（複数 Oracle）が、配信者にとって「次は別キャラで配信」の口実になり配信寿命を延ばした。
- Steam の DBG コミュニティで「次は何を遊ぶか」の文脈に乗りやすく、口コミ駆動でじわ伸びした典型的なローグライト・ロングテール挙動。

## 企画への示唆

- 飽和ジャンルに新メカニクスを差し込むとき、**構築単位そのものを置換する**（カード → ダイス）と、ジャンル文法を温存しつつ新味を出せる。
- ダメージ軸を二軸化して「敵にも自分にも作用しうる」ようにすると、リスクを取る構築選択が成立し戦術空間が広がる。
- 出目確率を「面数 / 6」で常時可視化すると、運ゲー印象を弱め期待値計算のゲームに変換できる。
- PC → コンソール → モバイルの段階展開を 2 年スパンで設計しておくと、同じ作品で 3 回のローンチ報道を踏める。
- ローグライトを成立させるには、**キャラクター（Oracle）ごとの初期解が違う**ことが配信者経済と相性が良く、ロングテール露出に直結する。

## 比較で見るべきタイトル

- Inscryption（DBG の語彙をジャンル外要素で再構築した同世代）
- Monster Train（DBG ローグライトの並列構築・パワーカーブ設計）
- Balatro（2024 年、確率と構築を別文法で再発明した同潮流）
