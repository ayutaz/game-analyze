## なぜ EXP 主分類なのか

スーパーマリオランのプレイヤーが繰り返し起動する動機は「片手で走り、跳ぶ気持ちよさを再現する」体験であり、収集や物語ではない。マリオは自動で右に走り続け、プレイヤーの入力はタップ一点に集約される。30 秒の電車待ちでも成立する短い操作ループで、ジャンプの長さ・空中アクション・コインの拾い方を最適化していく — この操作の磨き込み自体が娯楽である。

concept にある通り、買い切り型でガチャを排し、ワールドツアー／キノピオラリー／リミックス10 という三本のモードを並列で持つ。target はライト層とコンソールゲーマーで、いずれも「マリオの操作感そのもの」目当てに来る層。social_axis は非同期で、キノピオラリーで他プレイヤーのスコアやキノピオ獲得数を競うが、リアルタイム対戦ではないので、本質は「自分のジャンプ精度を高めるソロ体験 + 他者ランキングで揺さぶる」二層構造になっている。secondary に REW があるのは、キノピオ収集／きせかえ／きのこ王国の建築要素が長期リテンションを補強しているためで、主動機ではなく副次層に位置する。

## 体験の核 / 設計の発明

| 設計要素 | 内容 | 前世代（コンソール 2D マリオ）との差分 |
|---|---|---|
| 入力系 | タップのみ（長押しでジャンプ長を制御） | 十字キー＋ A/B の 2 軸操作を 1 軸に圧縮 |
| 進行 | オート右進行、停止不可 | 任意停止・逆走が可能 |
| ステージ長 | 1 ステージ 30〜90 秒 | 数分単位のステージ設計 |
| 失敗の救済 | バブル復活（コイン消費） | リトライ前提のミス即死 |
| 競争軸 | キノピオラリーで他プレイヤーのコイン獲得数を非同期比較 | スコアアタックは基本ローカル |
| 収益 | 前半無料 + フル $9.99 買い切り | パッケージ販売 |

設計の中核は「2D マリオの操作スキルを 1 軸入力に再翻訳する」点にある。タップの長押し時間 = ジャンプ放物線、空中タップ = 壁ジャンプ／空中スピン、という対応関係を作り、コンソール版で培われた「コイン取り＝ジャンプ軌道の最適化」というスキル要素を残した。非同期軸は対戦の心理的負荷を切り離し、ライト層が「他者の影」だけを感じる設計で、コミット圧を上げずに再起動動機を作る。

## バズ要因 / 人気の根拠

- iOS 配信初日に 285 万 DL を記録し、当時ピーク中だったポケモン GO の初日数字を上回った（任天堂初の本格スマホタイトルという報道文脈と相乗）
- 配信 4 日で 4,000 万 DL、2017 年 10 月までに全世界 2 億 DL を突破、最終的に 3.8 億 DL 超に到達（Sensor Tower）
- 国内 DL は約 2,000 万で全世界の約 10%（任天堂 2017 年 10 月時点公表）、マリオブランドの国内基盤の厚さを示す
- 累計収益約 96.6M ドルにとどまり、任天堂自身が「許容できる利益水準には達していない」と公言。DL 数 / 収益の乖離が「課金文化と買い切りの非整合」事例として業界文献に頻出している

## 企画への示唆

- 操作軸を 1 つに圧縮する代わりにタイミング表現（長押し・タップ位置・空中追加入力）を多層化すると、操作スキルの深度を維持できる
- 買い切り型をモバイルで通すと DL は爆発するが ARPU が崩れる：DL 規模指標と収益指標は別 KPI で監視する必要がある
- 非同期競争（フレンドのスコア / コイン数を「影」として提示）をソロランナーに重ねると、対戦の心理障壁を上げずにリテンションを伸ばせる
- 30〜90 秒のステージ尺と片手プレイ前提に絞ると、コアファン以外の「電車待ち層」が DAU に積み上がる — ただし課金導線を別途設計しないと収益化が遅延する
- 既存 IP のモバイル移植は「操作感の再翻訳」を行う方が、フル機能移植より評価が安定する（マリオは 1 軸化、ファイアーエムブレム ヒーローズはマップを 8×6 に縮小、という同じ翻訳パターンが任天堂内部で並走している）

## 比較で見るべきタイトル

- Temple Run / Subway Surfers：オートランナーの源流。マリオランはここに 2D 横スクロールの IP 文脈を重ねた
- ファイアーエムブレム ヒーローズ：同じ「コンソール文法のモバイル再翻訳」だが、課金モデルはガチャ採用で収益面の対照例
- Flappy Bird：1 タップ入力 + 短尺ループの極限化事例
- どうぶつの森 ポケットキャンプ：任天堂モバイル戦略のもう一つの軸（非同期 + 長期収益）
